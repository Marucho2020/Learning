2. Downcasting (ép kiểu từ cha xuống con):

	* Không được tự động thực hiện (không ép kiểu ẩn). Phải ép tường minh, 
		-> Why ? Một Parent không phải luôn luôn là một Child nên Java yêu cầu lập trình viên phải xác nhận bằng cách tự viết code ép kiểu tường minh 
				-> Điều này làm lập trình viên hiểu rõ được code đang việt và tránh các lỗi Logic 
				-> Nếu Java cho phép downcasting ẩn, bạn có thể gặp lỗi runtime nghiêm trọng khi cố gắng truy cập các class con không tồn tại trong đối tượng thực tế 
	
		## Ví dụ ép hợp lệ Downcasting 
		Parent parent = new Child();  // Upcasting (hợp lệ).
		Child child = (Child) parent; // Downcasting (phải tường minh).	
			
		
		## Ví dụ ép lỗi Downcasting 
				Child child = (Parent) (new Child());   // Lỗi biên dịch: Không thể gán Parent cho Child. phải sử dụng ép tường minh
