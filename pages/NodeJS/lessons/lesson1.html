# 2. What is Node.js? (Node.js Là Gì?)

	Tài liệu mô tả: Node.js là một runtime JavaScript miễn phí, mã nguồn mở, chạy trên nhiều nền tảng (Windows, Mac, Linux). Nó cho phép chạy JS ngoài browser, chủ yếu cho server-side. Xây dựng trên V8 engine của Chrome, Node.js tập trung vào các ứng dụng mạng scalable.	

	## Giải thích chi tiết:
		Runtime vs. Language: JavaScript là ngôn ngữ, nhưng browser (như Chrome) chỉ chạy JS ở client-side. Node.js là "môi trường chạy" (runtime) cho JS ở server-side, tương tự như Python interpreter cho Python code.

	## V8 Engine: 
		Đây là engine biên dịch JS thành machine code nhanh chóng. Google phát triển V8 cho Chrome, và Node.js dùng nó để đạt hiệu suất cao. Ví dụ: Node.js xử lý hàng nghìn kết nối đồng thời mà không tốn nhiều RAM.

	## Scalable Network Applications: 
		Node.js lý tưởng cho app cần xử lý nhiều I/O (input/output) như chat app, vì nó không chờ đợi (non-blocking). 


	## Ví dụ minh họa: Code đơn giản trong tài liệu: 
		console.log('Hello from Node.js!');

	Để chạy: Lưu vào file hello.js, mở terminal, gõ node hello.js. Kết quả: In ra "Hello from Node.js!" ở console.

# 3. Why Node.js? (Tại Sao Nên Dùng Node.js?)

	Node.js nổi bật với khả năng xử lý nhiều kết nối đồng thời mà không tốn tài nguyên, phù hợp cho:
	
		- Real-time apps (chat, game, collaboration).
		- APIs và microservices.
		- Data streaming (video, file upload).
		- Command-line tools.
		- Server-side web apps.

	## Giải thích chuyên sâu: 
		- Non-blocking, Event-driven Architecture: Không như PHP hoặc Java (blocking I/O), Node.js dùng event loop để xử lý sự kiện (events) mà không dừng chương trình. Điều này làm nó hiệu quả cho I/O-heavy tasks (như đọc file, query DB)
		
		- So sánh với Browser JS: Trong browser, JS xử lý events như click button. Node.js mở rộng ý tưởng này cho server, xử lý events như "request đến" hoặc "file đọc xong".
	
		- Ưu điểm thực tế: Netflix dùng Node.js cho streaming, LinkedIn cho backend – vì nó nhanh và dễ scale horizontally (thêm server).

	Mẹo: Nếu app của bạn nặng về CPU (như tính toán phức tạp), Node.js có thể không lý tưởng (single-threaded), nhưng bạn có thể dùng Worker Threads để multi-thread.	


# 4. Asynchronous Programming (Lập Trình Bất Đồng Bộ) 
	
	Node.js dùng async (non-blocking) để không chờ đợi tasks như đọc file hoặc query DB.
	Giải thích chi tiết:
	
		Blocking vs. Non-blocking: Blocking: Chương trình dừng chờ task xong (như synchronous code). Non-blocking: Tiếp tục chạy code khác trong lúc chờ.
	
		Callback Hell: Async thường dùng callbacks, nhưng có thể dẫn đến code lồng nhau phức tạp. Giải pháp: Sử dụng Promises hoặc async/await (ES6+).
		
		
	## 	Ví dụ
		
const fs = require('fs');

fs.readFile('myfile.txt', 'utf8', (err, data) => {
  if (err) {
    console.error('Error reading file: ' + err);
    return;
  }
  console.log('File content: ' + data);
});

console.log('Reading file... (this runs first!)');		
		
		
	## 	Phân tích:
		
		- require('fs'): Import module file system built-in.
		- readFile: Đọc file async, truyền callback (hàm chạy khi xong).
		- Lệnh console.log sau chạy ngay, không chờ đọc file – đó là non-blocking.
		- Nếu file tồn tại, callback in nội dung; nếu lỗi, in error.		
		

	## Phiên bản hiện đại với async/await:

const fs = require('fs/promises'); // Sử dụng promises version

async function readMyFile() {
  try {
    const data = await fs.readFile('myfile.txt', 'utf8');
    console.log('File content: ' + data);
  } catch (err) {
    console.error('Error: ' + err);
  }
}

readMyFile();
console.log('Reading file... (this still runs first!)');


# 5. What Can Node.js Do? (Node.js Có Thể Làm Gì?)

	- Web Servers: Tạo server HTTP.
	- File Operations: Đọc/ghi file qua fs.
	- Database Interaction: Kết nối MySQL/MongoDB.
	- APIs: Build REST/GraphQL.
	- Real-time: WebSockets cho live data.
	- CLI Tools: Như npm scripts.



	## Ví dụ Web Server:

const http = require('http');

http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello World!');
}).listen(8080);
Phân tích: Tạo server lắng nghe port 8080. Khi request đến, response "Hello World!".
Thực tế: Trong production, dùng Express.js để dễ quản lý routes, middleware.


# 6. What is a Node.js File? (File Node.js Là Gì?)

	- File .js chứa code chạy trên server.
	- Chạy bằng node filename.js.
	- Xử lý events (như requests).

	Giải thích: Không như HTML/JS browser (tự chạy khi load page), Node.js file cần khởi động thủ công. Dùng process.exit() để dừng nếu cần.
