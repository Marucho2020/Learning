# 2. Công thức hồi quy được viết lại:

	\text{Calorie_Burnage} = 0.3296 \times \text{Average_Pulse} + 346.8662

Thành phần,Giá trị,Ý nghĩa thực tế
Hệ số góc (Slope),0.3296,"Mỗi khi nhịp tim tăng 1, calo đốt cháy tăng 0.33"
Hệ số chặn (Intercept),346.8662,"Khi nhịp tim = 0, calo đốt cháy ≈ 347"


	Lưu ý: Nhịp tim = 0 là không thực tế (người chết rồi!), nhưng toán học vẫn cần hệ số chặn để đường thẳng "đúng vị trí".





# 3. Giải thích ý nghĩa hệ số:


Câu hỏi,Trả lời
Average_Pulse tăng 1 → Calorie_Burnage tăng bao nhiêu?,+0.3296 ≈ 0.33 calo
Average_Pulse = 0 → Calorie_Burnage = ?,346.8662 ≈ 347 calo


	Intercept không phải lúc nào cũng có nghĩa thực tế, nhưng luôn cần để mô hình chính xác.

# PHẦN 3: DỰ ĐOÁN BẰNG HÀM TRONG PYTHON

def Predict_Calorie_Burnage(Average_Pulse):
    return 0.3296 * Average_Pulse + 346.8662

print(Predict_Calorie_Burnage(120))  # → 386.4182
print(Predict_Calorie_Burnage(130))  # → 389.7142
print(Predict_Calorie_Burnage(150))  # → 396.3062
print(Predict_Calorie_Burnage(180))  # → 406.1942

	## 2. Intercept có thực tế không?
	
Trường hợp,Giải thích
Không thực tế,Nhịp tim = 0 → không thể xảy ra
Vẫn cần thiết,Giúp đường thẳng đi qua đúng vùng dữ liệu	
	
Calorie_Burnage
↑
400 |           o o o
    |         o     o
350 |       o         o
    |     o             o
300 |   o
    +-------------------→ Average_Pulse
      0  100  120  140  160	
	
	
→ Nếu không có intercept, đường thẳng có thể lệch xuống dưới, dự đoán sai.	
	
	
	
	
# 	3. Cách lấy hệ số tự động từ statsmodels (thay vì gõ tay)
import statsmodels.formula.api as smf

model = smf.ols('Calorie_Burnage ~ Average_Pulse', data=df).fit()

# Lấy hệ số tự động
slope = model.params['Average_Pulse']      # → 0.3296
intercept = model.params['Intercept']      # → 346.8662

print(f"Calorie_Burnage = {slope:.4f} * Average_Pulse + {intercept:.4f}")
