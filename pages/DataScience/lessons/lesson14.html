Lesson 14 -- Độ lệch chuẩn (Standard Deviation) và Hệ số biến thiên (Coefficient of Variation) -//


# 1. Độ lệch chuẩn (Standard Deviation) là gì?

	Định nghĩa: Độ lệch chuẩn (σ) đo lường mức độ phân tán của các giá trị trong tập dữ liệu so với giá trị trung bình (mean). Nó cho biết dữ liệu dao động nhiều hay ít quanh giá trị trung bình.

	Độ lệch chuẩn thấp: Các giá trị gần với trung bình, dữ liệu ít biến động.
	Độ lệch chuẩn cao: Các giá trị phân tán rộng, dữ liệu biến động lớn.

	Ý nghĩa: Trong phân tích dữ liệu, độ lệch chuẩn giúp đánh giá mức độ không chắc chắn (uncertainty). Ví dụ, nếu bạn dự đoán doanh thu dựa trên dữ liệu lịch sử, độ lệch chuẩn cao cho thấy doanh thu dao động mạnh, làm giảm độ tin cậy của dự đoán.

	Ký hiệu: Thường được biểu diễn bằng σ (sigma) trong thống kê dân số hoặc s trong mẫu.


	Ví dụ từ bài học:

		Với tập dữ liệu full_health_data (bao gồm các cột Duration, Average_Pulse, Max_Pulse, Calorie_Burnage, Hours_Work, Hours_Sleep), bài học sử dụng NumPy để tính độ lệch chuẩn:

import numpy as np
std = np.std(full_health_data)
print(std)

	Kết quả sẽ cho độ lệch chuẩn của từng cột, nhưng W3Schools không hiển thị số cụ thể vì dữ liệu lớn. Tuy nhiên, họ lưu ý rằng Duration, Calorie_Burnage, Hours_Work có độ lệch chuẩn cao hơn so với Max_Pulse, Average_Pulse, Hours_Sleep




# 2. Hệ số biến thiên (Coefficient of Variation)
	Định nghĩa: Hệ số biến thiên (CV) là tỷ lệ giữa độ lệch chuẩn và trung bình:


		$CV = \frac{\text{Standard Deviation}}{\text{Mean}}$

		CV thường được biểu diễn dưới dạng phần trăm (%).

	Ý nghĩa: CV chuẩn hóa độ lệch chuẩn, cho phép so sánh mức độ phân tán giữa các biến có đơn vị hoặc thang đo khác nhau. CV cao hơn nghĩa là biến đó có độ biến động tương đối lớn hơn.

	Ứng dụng: CV hữu ích khi so sánh dữ liệu từ các tập dữ liệu khác nhau, ví dụ, so sánh độ phân tán của nhịp tim (bpm) với lượng calo đốt (kcal).

import numpy as np
cv = np.std(full_health_data) / np.mean(full_health_data)
print(cv)


	Kết quả cho thấy Duration, Calorie_Burnage, Hours_Work có CV cao hơn, nghĩa là các biến này biến động tương đối lớn so với trung bình của chúng, trong khi Max_Pulse, Average_Pulse, Hours_Sleep ổn định hơn.

	Tính độ lệch chuẩn thủ công cho Max_Pulse

	Công thức độ lệch chuẩn (cho dân số, vì NumPy dùng ddof=0 mặc định):

	$$\sigma = \sqrt{\frac{\sum (x_i - \mu)^2}{n}}$$

	$x_i$: Giá trị từng phần tử.
	$\mu$: Giá trị trung bình.
	$n$: Số lượng phần tử.

	Bước 1: Tính trung bình ($\mu$) của Max_Pulse
		Max_Pulse: 120, 120, 130, 130, 140, 140, 145, 145, 150, 150. 

	137.

	Bước 2: Tính tổng bình phương khoảng cách đến trung bình
	$289 + 289 + 49 + 49 + 9 + 9 + 64 + 64 + 169 + 169 = 1057$	
		
	
	Bước 3: Tính độ lệch chuẩn
		 10.28.
	
	
	Bước 4: Tính hệ số biến thiên (CV)	
		 0.075 (hay 7.5%).
	
	
	Tính độ lệch chuẩn và CV cho Duration (để so sánh)

		Duration: 30, 30, 45, 45, 45, 60, 60, 60, 75, 75.
		Trung bình:52.5
		Tổng bình phương khoảng cách:506.25 (Tương tự cho các giá trị khác, tính nhanh: $506.25 \times 2 + 56.25 \times 3 + 56.25 \times 3 + 506.25 \times 2 = 2250$).
		$\sigma = \sqrt{\frac{2250}{10}} = \sqrt{225} = 15$		
		$CV = \frac{15}{52.5} \approx 0.286$ (hay 28.6%).

	Kết luận từ tính toán:
		- Max_Pulse: Độ lệch chuẩn ~10.28, CV ~7.5% → Phân tán thấp, nhịp tim tối đa khá ổn định.
		- Duration: Độ lệch chuẩn ~15, CV ~28.6% → Phân tán cao, thời gian tập luyện biến động nhiều.
		- Điều này phù hợp với nhận xét trong bài: Duration có độ lệch chuẩn và CV cao hơn Max_Pulse.


# 4. Minh họa bằng Python

import numpy as np

# Dữ liệu mẫu
data = {
    'Duration': [30, 30, 45, 45, 45, 60, 60, 60, 75, 75],
    'Max_Pulse': [120, 120, 130, 130, 140, 140, 145, 145, 150, 150]
}

# Tính độ lệch chuẩn
std_duration = np.std(data['Duration'], ddof=0)  # ddof=0 cho dân số
std_max_pulse = np.std(data['Max_Pulse'], ddof=0)
print("Std Duration:", std_duration)  # ~15.0
print("Std Max_Pulse:", std_max_pulse)  # ~10.28

# Tính hệ số biến thiên
cv_duration = std_duration / np.mean(data['Duration'])
cv_max_pulse = std_max_pulse / np.mean(data['Max_Pulse'])
print("CV Duration:", cv_duration)  # ~0.286
print("CV Max_Pulse:", cv_max_pulse)  # ~0.075
