Lesson 32 -- useEffect -//

# 1. What is useEffect?

	useEffect cho phép chạy code side effects sau render. Ví dụ: Fetch API, set timers, add event listeners. Callback chạy sau browser paint, tránh block UI.
	Insight chuyên gia: Side effects phải ở useEffect để giữ component pure (predictable). React chạy effects sau commit phase. Nếu không deps, chạy mọi render; với deps, chỉ khi deps thay đổi (shallow compare). Lưu ý: Effects async, không block render.


	Insight chuyên gia: useEffect là Hook linh hoạt nhất, nhưng dễ lạm dụng dẫn đến bugs (như infinite re-renders hoặc stale closures). Trong React 18+, nó hỗ trợ Concurrent Rendering, cho phép suspend effects. Best practice: Giữ effects focused (one per effect), dùng ESLint react-hooks/exhaustive-deps để warn missing deps. Nếu effect sync, dùng useLayoutEffect (chạy trước paint). Alternative cho complex logic: Custom Hooks hoặc thư viện như React Query cho data fetching. Nhược điểm: Async nature có thể gây race conditions; giải pháp: Cleanup hoặc abort controllers.
	
	
	
	
	
#2. Example: Timer cơ bản (với vấn đề)

	Ví dụ dùng setTimeout để đếm giây, nhưng gây infinite loop vì không deps – effect chạy mọi render, update state, trigger re-render.


import { useState, useEffect } from 'react';  // Import Hooks
import { createRoot } from 'react-dom/client';

function Timer() {
  const [count, setCount] = useState(0);  // State cho đếm

  useEffect(() => {  // Effect không deps
    setTimeout(() => {
      setCount((count) => count + 1);  // Update state sau 1s
    }, 1000);
  });

  return 
I've rendered {count} times!
;  // Render state
}

createRoot(document.getElementById('root')).render(
  
);


	useState(0): Khởi tạo count = 0.
	useEffect(() => { ... }): Callback chạy sau mỗi render. setTimeout delay 1s rồi update count (functional update để tránh stale value).
	Return JSX: Hiển thị count, re-render khi count thay đổi → loop vô tận.
	Render: Mount component.
	
	Insight chuyên gia: Vấn đề: Không deps → effect chạy quá thường xuyên. Trong production, có thể gây performance issues (nhiều timers chồng). Debug: Thêm console.log trong effect để thấy loop. Best practice: Luôn thêm deps array trừ khi intentional (hiếm).



# 3. Controlling When Effects Run: Dependency Array

	Deps array kiểm soát execution:

		- No deps: Chạy mọi render.
		- Empty []: Chạy chỉ lần đầu (mount).
		- Với values [dep1, dep2]: Chạy mount + khi deps thay đổi.


	## Mã code ví dụ (no deps):
		useEffect(() => {
		// Runs on every render
		});
	
	
	## Mã code ví dụ (empty array):
		useEffect(() => {
		// Runs only on the first render
		}, []);	
	
	
	## Mã code ví dụ (with deps): 
		
		useEffect(() => {
		// Runs on first render + deps changes
		}, [prop, state]);	
	
	Insight chuyên gia: Deps shallow compare (===), nên primitives OK, nhưng objects/arrays cần memoize (với useMemo) để tránh unnecessary runs. ESLint warn nếu quên deps (ví dụ: Nếu dùng variable trong effect, phải deps). Nếu deps thay đổi thường, refactor logic để minimize.
	
	
# 4. Example: Fix Timer với Empty Array	
	
	Thêm [] để effect chỉ chạy mount, tránh loop.
	
import { useState, useEffect } from 'react';
import { createRoot } from 'react-dom/client';

function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    setTimeout(() => {
      setCount((count) => count + 1);
    }, 1000);
  }, []);  // Empty deps: Chỉ mount

  return 
I've rendered {count} times!
;
}

createRoot(document.getElementById('root')).render(
  
);
	
	}, []);: Empty array → effect chỉ chạy lần đầu, nhưng vẫn loop vì setCount trigger re-render (nhưng effect không rerun). Thực tế ví dụ này vẫn chỉ tăng 1 lần (fix loop), nhưng nếu cần recurring, dùng setInterval (xem cleanup).
	
	Insight chuyên gia: Empty deps tương đương componentDidMount. Useful cho one-time setup (fetch initial data). Lỗi: Stale closures nếu deps thiếu (ví dụ: count trong callback là 0 forever). Giải pháp: Functional update setCount(prev => prev + 1).
	
	
# 	5. Example: Effect with Dependency
	
	Effect phụ thuộc count: Chạy khi count thay đổi, update calculation.
	
import { useState, useEffect } from 'react';
import { createRoot } from 'react-dom/client';

function Counter() {
  const [count, setCount] = useState(0);
  const [calculation, setCalculation] = useState(0);

  useEffect(() => {
    setCalculation(() => count * 2);  // Update dựa trên count
  }, [count]);  // Deps: count

  return (
    <>
      
Count: {count}

      
      
Calculation: {calculation}

    
  );
}

createRoot(document.getElementById('root')).render(
  
);	
	
	[count]: Effect rerun khi count thay đổi.
setCalculation(() => count * 2): Functional setter (không cần deps cho count vì closure).
	
	
	Insight chuyên gia: Useful cho derived state (nhưng ưu tiên compute trong render nếu sync: const calculation = count * 2;). Multiple deps: [dep1, dep2] – rerun nếu bất kỳ thay đổi. Best practice: Nếu effect fetch data, thêm abort signal để tránh race.	
	
	Fetch data với deps:
	
	useEffect(() => {
  const controller = new AbortController();
  fetch(`https://api.example.com/data/${id}`, { signal: controller.signal })
    .then(res => res.json())
    .then(setData);
  return () => controller.abort();  // Cleanup
}, [id]);  // Rerun khi id thay đổi
	
	
	
	
# 	6. Effect Cleanup

Return function từ effect để cleanup (unmount hoặc trước rerun). Useful cho clear timers, unsubscribe, remove listeners để tránh leaks.


import { useState, useEffect } from 'react';
import { createRoot } from 'react-dom/client';

function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    let timer = setTimeout(() => {  // Name timer để clear
      setCount((count) => count + 1);
    }, 1000);

    return () => clearTimeout(timer);  // Cleanup
  }, []);

  return 
I've rendered {count} times!
;
}

createRoot(document.getElementById('root')).render(
  
);

let timer = setTimeout(...): Lưu ID để clear.
return () => clearTimeout(timer): Chạy khi unmount hoặc deps thay đổi (trước effect rerun).


	Insight chuyên gia: Cleanup chạy sync, trước effect tiếp theo. Must-have cho subscriptions (WebSockets, event listeners). Ví dụ với interval (recurring timer):

useEffect(() => {
  const interval = setInterval(() => setCount(prev => prev + 1), 1000);
  return () => clearInterval(interval);
}, []);

	Lỗi: Quên cleanup → memory leaks (đặc biệt hot module reload). Best practice: Luôn cleanup resources created in effect.
