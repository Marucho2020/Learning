Multi-Agent Collaboration
Một Agent điều phối nhiều Agent khác (Supervisor Agent, Worker Agent), xu hướng mới của AWS và Anthropic. 


# 1. Bản chất là gì? 


	Nói ngắn gọn:
		Multi-Agent Collaboration = Chia một bài toán lớn thành nhiều bài toán nhỏ, giao cho nhiều AI Agent chuyên môn khác nhau xử lý, sau đó tổng hợp kết quả.

	Giống như trong công ty.
	Không ai vừa làm CEO, vừa kế toán, vừa luật sư, vừa lập trình.


		Thay vào đó

		CEO
		 │
		 ├── HR
		 ├── Accountant
		 ├── Lawyer
		 └── Engineer


	CEO chỉ:
		nhận yêu cầu
		chia việc
		tổng hợp kết quả

		Đó chính là Supervisor Agent.
		Các nhân viên là
		Worker Agents




	Một Agent thì sao?
	Một Agent phải tự làm

		Hiểu yêu cầu
		↓
		Lập kế hoạch
		↓
		Search
		↓
		Coding
		↓
		Tính toán
		↓
		Database
		↓
		Viết báo cáo



	Rất nhiều responsibility.

	Agent dễ bị
		context quá dài
		reasoning kém
		tool selection sai
		hallucination


	Multi-Agent sẽ là

               User

                 │

         Supervisor Agent

        ╱      │       ╲

 Search   Coding   Finance

 Agent     Agent    Agent

      ╲      │      ╱

      Result Aggregation

             │

            User

	Mỗi Agent chỉ làm đúng chuyên môn.

# 4. Supervisor Agent là gì?

	Supervisor giống Project Manager.
	Ví dụ
	
	Bạn nói
		Lập kế hoạch chuyến đi Nhật

	Supervisor nghĩ

		Task 1

		Tìm vé máy bay
		↓
		Task 2

		Tìm khách sạn
		↓
		Task 3
		Lập lịch trình
		↓
		Task 4

		Ước tính chi phí

	Sau đó gọi

		Flight Agent
		Hotel Agent
		Travel Agent
		Budget Agent


	Supervisor không biết
		vé máy bay.
		Supervisor biết
		"Ai nên làm?"
		Đây là bản chất.

# 9. Các kiểu Multi-Agent phổ biến

| Pattern             | Ý nghĩa                                  |
| ------------------- | ---------------------------------------- |
| Supervisor → Worker | Phổ biến nhất                            |
| Peer-to-Peer        | Các Agent trao đổi ngang hàng            |
| Planner → Executor  | Planner lập kế hoạch, Executor thực hiện |
| Debate Agents       | Hai Agent tranh luận rồi quyết định      |
| Hierarchical        | Nhiều tầng quản lý                       |
| Swarm               | Hàng chục Agent làm song song            |





# 10. Trade-off

	Ưu điểm
		✅ Reasoning tốt hơn
		✅ Context nhỏ hơn
		✅ Dễ scale
		✅ Chuyên môn hóa
		✅ Ít hallucination hơn

	Nhược điểm
		❌ Tăng latency do phải điều phối nhiều Agent.
		❌ Chi phí cao hơn (nhiều lần gọi mô hình).
		❌ Đồng bộ trạng thái (state) và chia sẻ ngữ cảnh khó hơn.
		❌ Thiết kế orchestration và xử lý lỗi phức tạp hơn.

# 15. Cách AWS ra đề

	AWS hiếm khi hỏi trực tiếp: "Multi-Agent là gì?"

	Thay vào đó, đề sẽ mô tả các dấu hiệu như:
		bài toán có nhiều lĩnh vực chuyên môn,
		cần mở rộng từng chức năng độc lập,
		muốn giảm hallucination,
		cần tái sử dụng các thành phần,
		nhiều công cụ hoặc nhiều nguồn dữ liệu khác nhau.

	Nếu xuất hiện các dấu hiệu này, đáp án thường nghiêng về kiến trúc Supervisor Agent + Worker Agents thay vì một Agent duy nhất làm mọi việc.
