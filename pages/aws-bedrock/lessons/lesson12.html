Observability & Evaluation
Theo dõi Agent bằng CloudWatch, Trace, đánh giá Accuracy, Latency, Cost và Debug workflow. 

#  Bản chất

	Rất nhiều người nghĩ:
		"Agent chạy được là xong."
		Đó là tư duy của người mới.



	Trong production, câu hỏi thực sự luôn là:
		Agent có trả lời đúng không?
		Agent mất bao lâu?
		Agent gọi bao nhiêu model?
		Agent gọi tool nào?
		Agent lỗi ở bước nào?
		Chi phí mỗi request bao nhiêu?
		Nếu người dùng báo lỗi thì debug thế nào?

	Đó chính là:
		Observability
	
	Nếu Monitoring chỉ trả lời:
		"Server còn sống."

	thì
	
	Observability trả lời:
		"Tại sao hệ thống lại hành xử như vậy?"
		Đây là khái niệm cực kỳ quan trọng của DevOps và cũng áp dụng nguyên vẹn cho AI Agent.

# Có 5 thứ luôn phải theo dõi

1. Accuracy (Sự chính xác)

	Đây là KPI quan trọng nhất.

	Ví dụ

		User hỏi
			Chi phí EC2 tháng này?

		Agent trả lời
			100 USD

		Thực tế
			98 USD
		Accuracy cao.


		Nếu
			500 USD
		Accuracy thấp.




	Accuracy không chỉ là đúng/sai.

	Có thể đo:
		Faithfulness  - sự chung thủy 
		Hallucination 
		Groundedness	- tính căn cứ 
		Answer relevance	- Trả lời mức độ liên quan
		Retrieval quality	-  chất lượng truy xuất 

	Đây đều là metric trong Evaluation.

# Evaluation (sự đánh giá)

		Evaluation
		=
		đánh giá chất lượng.
		Không phải monitoring.
		
	Monitoring
		Agent sống.

	Evaluation
		Agent giỏi không.
		
	
				
		Ví dụ
			1000 câu hỏi

		Agent A
			Accuracy
				92%
				
		Agent B
			95%
			
		Agent B tốt hơn.	
	
	
	## AWS hỗ trợ
			Model Evaluation
			Prompt Evaluation
			RAG Evaluation
			Safety Evaluation	
		
	
#	Trade-off
	
| Muốn           | Đổi lại                     |
| -------------- | --------------------------- |
| Accuracy cao   | Latency tăng                |
| Accuracy cao   | Cost tăng                   |
| Cost thấp      | Có thể Accuracy giảm        |
| Latency thấp   | Có thể Prompt ngắn hơn      |
| Nhiều Tool     | Workflow phức tạp           |
| Trace chi tiết | Tăng log storage và chi phí |
	

| Evaluation    | Monitoring    |
| ------------- | ------------- |
| Đúng không    | Có chạy không |
| Accuracy      | CPU           |
| Hallucination | Memory        |
| Relevance     | Error Rate    |

		Observability = Hiểu được nguyên nhân, không chỉ nhìn thấy triệu chứng.
