Knowledge Base (RAG)
Agent truy xuất tài liệu từ S3/OpenSearch để có context trước khi suy luận.


#1. Định nghĩa bản chất
		Knowledge Base (RAG - Retrieval-Augmented Generation) là kỹ thuật không nhét toàn bộ kiến thức vào LLM, mà đi tìm tài liệu liên quan trước, rồi mới đưa tài liệu đó vào Prompt để LLM trả lời.


	Nói đơn giản:
	LLM không nhớ tất cả.
		Khi có câu hỏi →
		đi tìm tài liệu liên quan (Retrieve) →
		đưa tài liệu đó vào Context
		→
		LLM mới Generate câu trả lời.

	Đó chính là RAG.

# 2. Tại sao phải dùng RAG?

	Ví dụ:

	Bạn hỏi ChatGPT:
		"Chính sách nghỉ phép của công ty ABC là gì?"

	LLM sẽ gặp vấn đề:
		chưa từng học dữ liệu đó
		hoặc dữ liệu mới cập nhật hôm qua
		hoặc tài liệu nội bộ

	=> Không biết.

	Nếu Fine-tune:
		rất tốn tiền
		phải train lại

	Trong khi thực tế chỉ cần:


	Search tài liệu
	↓
	Đưa tài liệu vào prompt
	↓
	LLM đọc
	↓
	Trả lời
	Đó là lý do RAG ra đời.

# 5. Amazon Bedrock Knowledge Base hoạt động như thế nào?
	Amazon Bedrock tự động làm gần như toàn bộ pipeline RAG.

	S3
	↓
	Parse Document
	↓
	Chunk
	↓
	Embedding
	↓
	Vector Database
	↓
	Search
	↓
	LLM

	Developer gần như không cần tự code.
	
	
	## Bước 1
	
	Đọc tài liệu

	Ví dụ
		PDF
		DOCX
		TXT
		HTML
			
	
	## Bước 2 
		Chunk

		Một PDF 500 trang
		↓
		chia thành
			Chunk 1
			Chunk 2
			Chunk 3
		...
			vì LLM không đọc cả cuốn sách mỗi lần.	
			
	
	
	
	
	## Bước 3
		Embedding

		Mỗi chunk
		↓
		chuyển thành vector.
			
	
		Text
		↓
		Embedding
		↓
		768 numbers	
			
	
	
	## Bước 4

# 9. Trade-off
	
| RAG                    | Fine-tuning                                |
| ---------------------- | ------------------------------------------ |
| Dữ liệu cập nhật ngay  | Phải train lại                             |
| Rẻ                     | Đắt                                        |
| Không thay đổi model   | Thay đổi trọng số model                    |
| Tốt cho tài liệu       | Tốt cho phong cách hoặc tác vụ chuyên biệt |
| Có thể trích dẫn nguồn | Khó biết câu trả lời dựa trên tài liệu nào |
	
	
	
	
# 	10. So sánh các khái niệm dễ nhầm
	
	
| Khái niệm          | Mục đích                      |
| ------------------ | ----------------------------- |
| Prompt Engineering | Viết Prompt tốt hơn           |
| RAG                | Tìm tài liệu rồi mới hỏi LLM  |
| Fine-tuning        | Huấn luyện lại model          |
| Agent              | Quyết định dùng Tool nào      |
| Knowledge Base     | Kho tài liệu để RAG truy xuất |
	
	
	Mối quan hệ:
		Knowledge Base là nguồn dữ liệu.
		RAG là kỹ thuật truy xuất dữ liệu từ Knowledge Base rồi đưa vào Prompt.
		Agent có thể sử dụng Knowledge Base như một trong nhiều công cụ (tools) của mình.
			
			
	
	
## 	11. Cách AWS ra đề 
	
	
	
	Dạng 1
		Dữ liệu thay đổi liên tục.
		→ Không Fine-tune.
		→ Dùng Knowledge Base.

	Dạng 2

		Có hàng triệu tài liệu PDF trong S3.
		→ Knowledge Base.

	Dạng 3

		Muốn chatbot trả lời dựa trên tài liệu nội bộ.
		→ Knowledge Base.

	Dạng 4

		Muốn giảm Hallucination.
		→ RAG.

	Dạng 5

		Muốn Agent đọc tài liệu trước rồi mới quyết định.
		→ Agent + Knowledge Base.
			
	
# 	12. Những bẫy AWS rất thích hỏi
	
| Bẫy                                              | Đáp án đúng                                  |
| ------------------------------------------------ | -------------------------------------------- |
| Dữ liệu thay đổi liên tục nhưng chọn Fine-tuning | Sai, nên dùng RAG                            |
| Có PDF trong S3 nhưng không dùng Knowledge Base  | Sai                                          |
| Muốn cập nhật tài liệu ngay nhưng train model    | Sai                                          |
| Nghĩ Agent tự biết dữ liệu công ty               | Sai, Agent cần Knowledge Base hoặc tool khác |
| Nghĩ OpenSearch là LLM                           | Sai, OpenSearch chỉ lưu và tìm kiếm vector   |
	


# 13. Ghi nhớ để đi thi

	Knowledge Base = Kho tài liệu.
	RAG = Quy trình tìm tài liệu → thêm vào Context → LLM trả lời.
	Agent = Bộ điều phối, có thể sử dụng Knowledge Base như một công cụ để lấy ngữ cảnh trước khi suy luận.

	Nếu đề có các dấu hiệu:
		tài liệu nội bộ,
		PDF,
		S3,
		dữ liệu thay đổi thường xuyên,
		giảm hallucination,
		enterprise search,

	→ Đáp án gần như luôn xoay quanh Amazon Bedrock Knowledge Base + RAG.
