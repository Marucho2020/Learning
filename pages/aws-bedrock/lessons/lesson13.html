Agentic System Design
Thiết kế kiến trúc doanh nghiệp sử dụng Agent: phân chia nhiệm vụ, Human-in-the-loop, Retry, Fallback, Event-driven AI. 

	Đây là một trong những chủ đề quan trọng nhất nếu bạn muốn xây dựng hệ thống AI thật sự trên AWS, đặc biệt khi sử dụng Amazon Bedrock Agents, Multi-Agent Collaboration, EventBridge, Step Functions, Lambda.

	Đa số người mới đều nghĩ:
		Agent = Chatbot biết gọi tool.

	Đây là hiểu sai.
		Trong doanh nghiệp,

		Agent không phải chatbot. Agent là một worker (nhân viên) có khả năng tự lập kế hoạch, sử dụng công cụ, giao tiếp với agent khác và xử lý công việc.



# I. Bản chất của Agentic System

	Agentic System là một kiến trúc mà:
		Một bài toán lớn được chia thành nhiều nhiệm vụ nhỏ.
		Mỗi nhiệm vụ được giao cho một Agent chuyên trách.
		Giống hệt công ty ngoài đời.
		Không có CEO nào tự làm tất cả.


	Khách hàng hỏi

		"Tôi muốn hoàn tiền đơn hàng."
		Trong công ty sẽ không có một người làm hết.
		Thay vào đó


		Customer
		↓
		Customer Support
		↓
		Finance
		↓
		Order System
		↓
		Inventory
		↓
		Notification

		AI Agent cũng hoạt động y hệt.

# III. Task Decomposition		
		
			
	Đây là kỹ năng quan trọng nhất.
	Một Agent không nên làm nhiều việc.		
			
	Ví dụ sai			
		HR Agent
		Làm:
		Payroll
		Recruitment
		Interview
		Leave
		Training		
	Quá nhiều.				
		
		
		
	Thiết kế đúng

		Payroll Agent
		Recruitment Agent
		Interview Agent
		Leave Agent
		Một Agent = một domain.

	Đây chính là Single Responsibility Principle áp dụng cho AI.		
	AWS rất thích thiết kế này.

# Những bẫy AWS hay gài


| Bẫy                                             | Vì sao sai                                                      |
| ----------------------------------------------- | --------------------------------------------------------------- |
| Một Agent làm mọi việc                          | Khó mở rộng, khó bảo trì, vi phạm Single Responsibility         |
| AI tự động thực hiện mọi hành động quan trọng   | Thiếu Human-in-the-loop, rủi ro cao                             |
| Retry cho mọi lỗi                               | Có lỗi không thể phục hồi (Permission Denied, Validation Error) |
| Không có Fallback                               | Hệ thống dễ dừng khi model hoặc API gặp sự cố                   |
| Polling liên tục để kiểm tra dữ liệu mới        | AWS ưu tiên Event-driven với EventBridge/S3 Events              |
| Dùng Knowledge Base để lưu trạng thái hội thoại | Sai bản chất; trạng thái hội thoại thuộc Memory/Session         |



# XIII. Tóm tắt để nhớ lâu

| Khái niệm          | Hình dung ngoài đời                                |
| ------------------ | -------------------------------------------------- |
| Supervisor Agent   | Quản lý dự án (PM) giao việc cho nhân viên         |
| Task Decomposition | Chia một dự án lớn thành nhiều đầu việc nhỏ        |
| Human-in-the-loop  | Hồ sơ cần sếp ký trước khi thực hiện               |
| Retry              | Gọi lại khi đầu dây bên kia bận hoặc mất kết nối   |
| Fallback           | Kế hoạch B khi kế hoạch A thất bại                 |
| Event-driven AI    | Chuông báo vang lên thì nhân viên bắt đầu làm việc |
| Memory             | Sổ ghi chú tạm trong cuộc họp                      |
| Knowledge Base     | Thư viện tài liệu của công ty                      |
