# Trusted Advisor là gì?


	Định nghĩa bản chất
		Trusted Advisor là một dịch vụ phân tích (Advisor) giúp kiểm tra toàn bộ tài khoản AWS của bạn và đưa ra các khuyến nghị (recommendations) để hệ thống:
		Tiết kiệm chi phí (Cost Optimization)
		Tăng hiệu năng (Performance)
		Tăng tính bảo mật (Security)
		Tăng khả năng chịu lỗi (Fault Tolerance)
		Kiểm tra giới hạn dịch vụ (Service Quotas)
		Kiểm tra vận hành (Operational Excellence)

	Điểm quan trọng nhất:
		Trusted Advisor không sửa lỗi giúp bạn.

	Nó chỉ giống như một chuyên gia AWS đứng bên cạnh xem hệ thống và nói:
		"EC2 này đang nhàn rỗi."
		"S3 Bucket này đang public."
		"Bạn sắp hết quota."
		"Nên bật Multi-AZ."	
	Quyết định sửa hay không vẫn là của bạn.			
	
	
# Bản chất thật

	Nhiều người nghĩ:
		Trusted Advisor = Monitoring	
		
		
		Sai.
			Nó không monitor realtime như CloudWatch.	
			
		Nó giống:			
			
			CloudWatch
			↓
			Thu thập Metric
			↓
			Bạn tự phân tích

			------------------------
			Trusted Advisor
			↓
			AWS tự phân tích
			↓
			Đưa Recommendation	
				
				
			Nó sử dụng rất nhiều AWS Best Practices.
			Có thể hiểu đơn giản:
			Trusted Advisor = AWS Expert System.	
				
				
# 	Trusted Advisor kiểm tra những gì?
	
| Category               | Kiểm tra                                           |
| ---------------------- | -------------------------------------------------- |
| Cost Optimization      | EC2 nhàn rỗi, EBS không dùng, Elastic IP không gắn |
| Security               | MFA, S3 Public, Security Group mở 0.0.0.0/0        |
| Performance            | EBS throughput, CloudFront Optimization            |
| Fault Tolerance        | Multi-AZ, Backup                                   |
| Service Quotas         | Sắp đạt giới hạn EC2, VPC, IAM...                  |
| Operational Excellence | Best Practices vận hành                            |
	
	
# 	Mapping với dịch vụ AWS

| Trusted Advisor kiểm tra | Dịch vụ        |
| ------------------------ | -------------- |
| Idle EC2                 | EC2            |
| Unattached EBS           | EBS            |
| Idle Load Balancer       | ELB            |
| Public Bucket            | S3             |
| MFA                      | IAM            |
| Quota                    | Service Quotas |
| Multi-AZ                 | RDS            |
| Route53 Health           | Route53        |
| Snapshot                 | EBS            |
	
	
	
#Ví dụ Cost Optimization

		Bạn tạo
		t3.large
		24/24
		CPU
		2%
		Trusted Advisor sẽ báo:
			Underutilized EC2 Instance

		Gợi ý:
			Stop
			Resize
			Terminate
			Không tự làm.	
			
	
	
# Ví dụ Security 	
		
	Bạn tạo
		S3
		Public Read

	Trusted Advisor:

		Warning
		Bucket is publicly accessible.	
			
		
# 	Ví dụ Fault Tolerance
		
	Bạn có:
		Single-AZ RDS

	Trusted Advisor:
		Recommendation
		Enable Multi-AZ	
	
	



# Ví dụ Service Quotas


Giả sử quota:

	EC2
	Running On-Demand
	Limit
	20

	Bạn đang chạy
		18

	Trusted Advisor:

		Warning
		Approaching Service Limit

	Đây là cảnh báo để tránh việc deploy thất bại do hết quota.



# Ví dụ Operational Excellence 


	Ví dụ:
		Không backup
		Không tagging
		Không logging

	Trusted Advisor sẽ gợi ý theo AWS Well-Architected Best Practices.


# So sánh với CloudWatch 

| Trusted Advisor         | CloudWatch       |
| ----------------------- | ---------------- |
| Phân tích Best Practice | Thu thập Metrics |
| Recommendation          | Monitoring       |
| Không realtime          | Realtime         |
| Không Alarm             | Alarm            |
| Không sửa               | Không sửa        |
| AWS phân tích           | Bạn tự phân tích |



# So sánh với AWS Config

| Trusted Advisor   | AWS Config             |
| ----------------- | ---------------------- |
| Best Practice     | Compliance             |
| Recommendation    | Rule Evaluation        |
| Tư vấn            | Audit                  |
| Không lưu lịch sử | Lưu lịch sử thay đổi   |
| Tổng quan         | Chi tiết từng Resource |



# Trade-off

	Ưu điểm
		Dễ dùng
		Không cần tự phân tích
		Theo AWS Best Practice
		Giúp giảm chi phí
		Tăng bảo mật
		Tăng độ sẵn sàng

	Nhược điểm

		Không realtime như CloudWatch
		Không thay thế AWS Config
		Không tự sửa lỗi
		Một số kiểm tra yêu cầu gói AWS Support cao hơn

# Cách AWS ra đề
	AWS hiếm khi hỏi trực tiếp "Trusted Advisor là gì?". Thay vào đó, đề thường mô tả một nhu cầu:

		Giảm chi phí: tìm EC2 nhàn rỗi, EBS không dùng, Elastic IP dư thừa.
		Nâng cao bảo mật: phát hiện S3 public, IAM chưa bật MFA.
		Tăng độ tin cậy: khuyến nghị bật Multi-AZ, kiểm tra backup.
		Theo dõi quota: cảnh báo sắp chạm giới hạn dịch vụ.

	Nếu câu hỏi nhấn mạnh "AWS Best Practices", "recommendations", "cost optimization", "service limit checks" thì đáp án rất thường là Trusted Advisor.
