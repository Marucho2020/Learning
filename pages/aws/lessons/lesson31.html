Lesson 31 -- Cloud Practitioner --Versioning + Lifecycle + Replication combo -//


# 🧠 1. Bản chất tổng hợp (nói thẳng) 


| Feature     | Bản chất thật                     |
| ----------- | --------------------------------- |
| Versioning  | giữ lịch sử (rollback)            |
| Lifecycle   | tiết kiệm tiền (auto move/delete) |
| Replication | chống chết region (DR)            |



		👉 3 cái này KHÔNG trùng nhau:
			Versioning = safety (logic)
			Lifecycle = cost (money)
			Replication = survival (infra)



# 🔥 2. Mối quan hệ giữa 3 cái (cực quan trọng)


	👉 Điều nhiều người không biết:
		❗ Replication REQUIRE Versioning
		Không có versioning → replication không chạy
		👉 Đây là bẫy cực hay ra


	❗ Lifecycle hoạt động trên từng version
		có thể:
		xóa version cũ
		chuyển version cũ sang Glacier




	👉 Tổng hợp:

		Versioning → tạo nhiều version
		Lifecycle → dọn dẹp version đó
		Replication → copy version sang nơi khác


# 🧪 3. Scenario tổng hợp

	Scenario 1 (classic SAA)

		Công ty cần:

			bảo vệ dữ liệu khỏi xóa nhầm
			DR cross-region
			tối ưu cost cho data cũ

		👉 Solution chuẩn:

			Enable Versioning
			Enable Replication
			Add Lifecycle

		👉 Nếu thiếu 1 cái → sai



	## Scenario 2 (bẫy)
		Data cần replicate sang region khác nhưng KHÔNG cần giữ version
		👉 Nhiều người chọn replication ❌

		👉 Sai:
		→ replication bắt buộc versioning
		
		
		
	## Scenario 3 (bẫy cost) 	
		
		Bucket có versioning → chi phí tăng nhanh
			👉 Đáp án:
			→ Lifecycle để delete old versions		
					
				
	## Scenario 4 (bẫy delete)
	
		User xóa file → không muốn mất dữ liệu
			👉 Versioning là đủ
			👉 KHÔNG cần replication	
			
			
	
# 🔗 4. Mapping AWS 	
	
| Requirement       | Solution                 |
| ----------------- | ------------------------ |
| rollback          | Versioning               |
| cost optimization | Lifecycle                |
| DR                | Cross-Region Replication |
| compliance        | Object Lock              |

	
	
	
# 	🔑 5. Keyword mapping
	
	
| Keyword                  | Chọn        |
| ------------------------ | ----------- |
| accidental delete        | Versioning  |
| after X days             | Lifecycle   |
| cross-region DR          | Replication |
| reduce cost              | Lifecycle   |
| backup in another region | Replication |
	
	
	
# 	⚠️ 6. Bẫy cực mạnh
	
	❌ Bẫy 1: Replication ≠ Backup hoàn toàn
		replication:
		async
		copy lỗi vẫn replicate

	👉 không phải backup “an toàn tuyệt đối”	



	❌ Bẫy 2: Versioning ≠ DR
		cùng region → vẫn chết nếu region down


	❌ Bẫy 3: Lifecycle xóa nhầm

		👉 nếu config sai:
		xóa luôn version cần giữ


	❌ Bẫy 4: Delete propagation

		👉 delete marker cũng replicate
		→ xóa bên A → B cũng “mất”


# ⚖️ 7. So sánh tổng hợp

| Feature       | Versioning  | Lifecycle   | Replication  |
| ------------- | ----------- | ----------- | ------------ |
| Mục đích      | rollback    | giảm cost   | DR           |
| Trigger       | action      | time        | async        |
| Data location | same bucket | same bucket | cross region |
| Cost impact   | tăng        | giảm        | tăng         |
| Complexity    | thấp        | trung       | cao          |


# 🧠 8. Cách AWS ra đề


	🧪 Example 1

		Công ty cần:

		bảo vệ khỏi xóa nhầm
		replicate sang region khác
		tự động archive data sau 90 ngày

		👉 Đáp án:

			Versioning
			Replication
			Lifecycle

		👉 thiếu 1 cái = sai



	🧪 Example 2 (trap logic)

		Chỉ cần rollback file

		👉 Versioning là đủ
		👉 chọn thêm replication = sai



	🧪 Example 3 (bẫy cost + DR)

			DR + tối ưu chi phí
			👉 Đúng:
				Replication + Lifecycle


#🎯 9. Trade-off tổng hợp 

| Yếu tố     | Trade-off                     |
| ---------- | ----------------------------- |
| Cost       | tăng mạnh nếu không lifecycle |
| Complexity | cao                           |
| Safety     | cao                           |
| Latency    | replication có delay          |

				
				
# 🧠 10. Tóm tắt 


	👉 Nhớ 3 dòng này:

		Rollback → Versioning
		Tiết kiệm tiền → Lifecycle
		Chống chết region → Replication

	👉 Và câu quan trọng nhất:

		Replication cần Versioning
