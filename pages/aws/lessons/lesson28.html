# I. Giải thích bản chất


	Object Storage = lưu file dạng “object”, không có cấu trúc thư mục thật
		Mỗi file = 1 object
		Object = data + metadata + key (ID duy nhất)
		Không có folder thật → chỉ là prefix (giả thư mục)

	👉 Nói thẳng:
		Object Storage không phải file system → nó là key-value storage cho file




	## So sánh nhanh 

		| Loại storage        | Bản chất           | Dùng khi nào            |
		| ------------------- | ------------------ | ----------------------- |
		| Object Storage (S3) | key-value (file)   | ảnh, video, backup      |
		| Block Storage (EBS) | ổ cứng             | OS, database            |
		| File Storage (EFS)  | shared file system | nhiều server dùng chung |

		👉 Sai 1 câu là trượt mindset luôn


# II. Ví dụ thực tế

	## Ví dụ đời thường 

		Bạn upload ảnh Facebook:

			Ảnh = object data
			URL = key
			Metadata = userId, timestamp

			👉 Facebook không lưu ảnh bằng kiểu “folder/user123/img.png”
			→ nó lưu dạng key kiểu:
			user123/2026/04/img_abc123.jpg
			(chỉ là string, không phải folder thật)




	## Ví dụ system design
	
		Một app upload video
	
			Yêu cầu:

				- Scale vô hạn
				- Không mất dữ liệu
				- Truy cập từ internet
				- Solution chuẩn:
				- Upload → S3
				- Serve → CloudFront
				- Metadata → DynamoDB
			👉 Vì:

				- S3 = durable (11 số 9)
				- Không cần quản lý server	
	
	
# 	III. Mapping với AWS
		
	| Khái niệm      | AWS Service |
	| -------------- | ----------- |
	| Object Storage | S3          |
	| CDN            | CloudFront  |
	| Metadata DB    | DynamoDB    |
	| Event trigger  | S3 → Lambda |
		
		
	
# IV. Các từ khóa trong đề thi	
	
	- “Highly durable”
	- “Store large amount of unstructured data”
	- “Static website hosting”
	- “Backup & archive”
	- “Media files (image, video)”
	- “11 9 durability”	
	
	
# 	V. Các bẫy (trap) cực hay gặp
	
	Trap 1: Nhầm với EBS

	Đề:
		cần storage cho EC2 để chạy database
		👉 Chọn S3 = sai
		✔ Đúng: EBS
		→ vì DB cần block storage	
		
	

	Trap 2: Nhầm với EFS

		Đề:

		nhiều EC2 cần share file

		👉 Chọn S3 = sai

		✔ Đúng: EFS
			→ vì cần file system



	Trap 3: tưởng S3 có folder thật
	
		👉 Không có
			/images/2026/abc.jpg = key string thôi



	Trap 4: latency

		👉 S3 không dành cho:
		low latency read/write (DB)



# VI. Trade-off

| Ưu điểm         | Nhược điểm               |
| --------------- | ------------------------ |
| Scale vô hạn    | Không sửa file từng phần |
| Durable cực cao | Latency cao hơn EBS      |
| Rẻ              | Không mount như disk     |
| Serverless      | Không dùng cho DB        |



# VII. Cách AWS ra đề

	Pattern 1: hỏi loại storage
		“store images, videos, backups”
		→ chọn S3



	Pattern 2: hỏi cost + scale
		“cost-effective + large scale + no management”
		→ S3

	Pattern 3: hỏi kiến trúc	
		upload file → xử lý → lưu trữ
		→ S3 + Lambda


	Pattern 4: đánh lừa bằng EC2

		App chạy EC2 nhưng storage là gì?
			👉 Đừng auto chọn EBS
			→ nếu là file/media → S3



# VIII. Mini Scenario

	Câu hỏi:

	Một công ty cần:

		Lưu trữ hàng triệu ảnh
		Truy cập qua internet
		Không cần chỉnh sửa từng byte
		Chi phí thấp
	👉 Chọn gì?

		Phân tích:
			Nhiều file → Object storage
			Không edit → OK với S3
			Public access → S3 + CloudFront

		✔ Đáp án: S3
		S3 = “dump file vào đó, scale vô hạn, đừng dùng cho DB”

# 	I. 本質
	Object Storage = ファイルを「オブジェクト」として保存する仕組み		
		
		- 各ファイル = Object
		- Object = データ + メタデータ + Key
		- フォルダは実体なし（prefixのみ）		
				
	👉 本質：
		Object Storage = ファイルのためのKey-Valueストレージ		
		
		
#	II. ストレージ比較
		
| 種類  | 特徴             | 用途           |
| --- | -------------- | ------------ |
| S3  | Object Storage | 画像・動画・バックアップ |
| EBS | Block Storage  | OS・DB        |
| EFS | File Storage   | 複数EC2共有      |
		
		
		
		
# 	III. 実例	
		
		
	システム設計

		動画アップロードサービス：
			- 保存 → S3
			- 配信 → CloudFront
			- メタデータ → DynamoDB

		理由：
			- 高耐久性（11 9）
			- スケーラブル
			- サーバ不要		
		
		
		
# IV. 試験キーワード		
			
	高耐久性（11 9 durability）
	非構造データ
	静的Webホスティング
	バックアップ
	メディアファイル		
		
		
		
# V. よくある罠		
		
		
	罠1：EBSと混同

		DB用途 → S3 ❌ → EBS ⭕

	罠2：EFSと混同

		共有ファイル → S3 ❌ → EFS ⭕

	罠3：フォルダがあると思う
		→ 実際はただの文字列

	罠4：低レイテンシ用途

		→ S3は不向き		
		
		
		
VII. AWSの出題パターン 		
		
		
	パターン1
		画像・動画保存 → S3

	パターン2
		低コスト + 大規模 → S3

	パターン3
		アップロード処理 → S3 + Lambda

	パターン4
		EC2が出ても騙されるな
		→ ファイルならS3
