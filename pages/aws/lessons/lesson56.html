#  VPC là gì?（Virtual Private Cloud）

	VPC = Mạng riêng ảo của bạn bên trong AWS.



	Khi tạo AWS Account:
		AWS có một hệ thống mạng khổng lồ
		Hàng triệu khách hàng dùng chung hạ tầng

	VPC là cơ chế để:
		Tách riêng một phần mạng cho riêng bạn.


| Thực tế                | AWS               |
| ---------------------- | ----------------- |
| Tòa nhà văn phòng      | AWS Region        |
| Tầng lầu               | Availability Zone |
| Công ty thuê văn phòng | VPC               |
| Phòng ban              | Subnet            |


	Ví dụ:

	Bạn thuê văn phòng.

	Bạn quyết định:
		IP nào được dùng
		Ai được vào
		Ai được ra Internet
		Server nào nói chuyện với server nào

	Đó chính là VPC.



# 2. AWS tạo VPC để giải quyết vấn đề gì?


	## Không có VPC:
		EC2 khách hàng A
		EC2 khách hàng B
		EC2 khách hàng C

	nằm chung mạng

	=> Thảm họa bảo mật.



	## Có VPC:

		Customer A
		VPC A
		10.0.0.0/16

		Customer B
		VPC B
		172.31.0.0/16
	Tách biệt hoàn toàn.



# 3. Thành phần bên trong VPC
	VPC = Container mạng
	Bên trong có:
VPC
├── Subnet
├── Route Table
├── Internet Gateway
├── NAT Gateway
├── Security Group
├── NACL
├── EC2
├── RDS
├── ELB



# 4. CIDR Block là gì?
	Khi tạo VPC:
		10.0.0.0/16
		
	AWS hỏi:
		IP range của mạng này là gì?
	
	/16 nghĩa là:
	
		10.0.0.0
		↓
		10.0.255.255	
		Khoảng:	65,536 IP
		
	
	Ví dụ công ty:
		Tokyo Office
		
		10.0.x.x
		AWS cũng y hệt.
	

# 5. Subnet là gì? 

	Nhiều người học sai.
	Họ nghĩ:
		VPC = server
		Subnet = server
	Sai.

		VPC = thành phố
		Subnet = quận


	Ví dụ:
		VPC
			10.0.0.0/16

	Chia:
		Public Subnet
			10.0.1.0/24

		Private Subnet
			10.0.2.0/24



# 6. Public Subnet và Private Subnet 

	## Public Subnet
		Có Route tới Internet Gateway

	0.0.0.0/0
	↓
	IGW
	Có thể ra Internet.

	Ví dụ:
		ALB
		Bastion
		Web Server	
	
	
	
	
	
	## Private Subnet
		Không route trực tiếp tới Internet.
		Ví dụ:
	
		RDS
		Application Server
		Redis
	
	
	## Bẫy thi 	
	
	Có Public IP
	=
	Public Subnet	

	Sai.
		Điều kiện Public Subnet:
			Route Table
			có route tới IGW
			
		Mới là Public Subnet.
		
		
		
# 	7. Internet Gateway (IGW)	
		
	Bản chất:	Cổng nối VPC với Internet	
	Không có IGW:
		
		EC2
		↓
		Internet

		Không đi được		
		
		
# 8. NAT Gateway 
	Khái niệm cực hay bị hỏi.


	Ví dụ:
		Private EC2

	cần:
		apt update
		yum update
		download package


	Không muốn Internet truy cập ngược vào.


	Giải pháp:
		Private EC2
		↓
		NAT Gateway
		↓
		Internet


	Kết quả:
	✅ Đi ra được
	❌ Internet không vào được



	## AWS rất thích hỏi 
		
			
	Công ty cần:
		Download update
		Nhưng không public server

	Đáp án:
		NAT Gateway		
		
		
		
		
# 	9. Route Table	
	Bản chất:	
		Bảng chỉ đường
		
	Ví dụ:
		Đi đâu?		
		
		
	Route:

		10.0.0.0/16
		local

	Nghĩa là:
		Trong VPC	
		
		
		
	Route:
		0.0.0.0/0
		IGW

	Nghĩa là:
		Internet		
		
		
		
		
		
# 	10. Security Group


	AWS gọi:
		Virtual Firewall

	Nhưng hiểu sâu hơn:
		Firewall gắn vào ENI (card mạng của instance)



	Ví dụ:
		Allow 80
		Allow 443
		Allow 22

	Đặc điểm cực quan trọng:
		✅ Stateful

	Nghĩa là:
		Request được phép

	thì:
		Response tự động được phép





# 11. Network ACL (NACL)
	Một lớp firewall khác.


| Security Group | NACL         |
| -------------- | ------------ |
| Stateful       | Stateless    |
| Instance Level | Subnet Level |
| Allow Only     | Allow + Deny |
	
	AWS rất thích hỏi.	
		
	Ví dụ:
		Chặn IP cụ thể
		Security Group làm khó.
		NACL làm dễ.		
		
		
	## So sánh SG và NACL 

| Tiêu chí        | SG       | NACL              |
| --------------- | -------- | ----------------- |
| Level           | Instance | Subnet            |
| Stateful        | Có       | Không             |
| Deny Rule       | Không    | Có                |
| Thi thường chọn | SG       | NACL nếu cần deny |



# Từ khóa thi SAA 



| Từ khóa            | Nghĩ ngay      |
| ------------------ | -------------- |
| Internet access    | IGW            |
| Outbound only      | NAT Gateway    |
| Stateful firewall  | SG             |
| Stateless firewall | NACL           |
| Public-facing      | Public Subnet  |
| Database security  | Private Subnet |
| CIDR               | IP Range       |
| Route              | Định tuyến     |
| Isolation          | VPC            |
