# 1. Bản chất thật sự của Subnet là gì?

Nếu chỉ nhớ 1 câu:
	Subnet = chia nhỏ mạng VPC thành nhiều vùng mạng logic để kiểm soát routing, security và khả năng truy cập.


	Nhiều người mới học thường nghĩ:
		VPC = mạng
		Subnet = thư mục con trong mạng

	=> Sai bản chất.

	Thực tế:
		VPC chỉ là một vùng địa chỉ IP lớn.


	Ví dụ:

	VPC
		10.0.0.0/16
	AWS cấp cho bạn:
		10.0.0.0 ~ 10.0.255.255

	Tổng:
		65,536 IP
		Nhưng bạn không thể ném toàn bộ EC2, RDS, ALB vào cùng một đống.
	Đó chính là Subnet.

	Tương tự:
		Public Server

		không nên ở cùng subnet

# 4. Public Subnet vs Private Subnet

	Đây là câu AWS cực thích hỏi.
	Sai lầm phổ biến
	Nhiều người nghĩ:
	Public Subnet vì có Public IP

	Sai.

	AWS định nghĩa:
	Public hay Private phụ thuộc Route Table.
	
	
	
# 	Public Subnet
	Có route:	
	
0.0.0.0/0
    ↓
Internet Gateway	
	
	
Ví dụ:

Destination	Target
10.0.0.0/16	local
0.0.0.0/0	IGW	
	
	Có đường ra Internet.
	=> Public Subnet
	
	
	
# Private Subnet
	Không có route tới IGW.

| Destination | Target |
| ----------- | ------ |
| 10.0.0.0/16 | local  |
	=> Private Subnet



AWS Trap số 1

	AWS hỏi:

		Một EC2 có Public IP. Nó nằm trong Private Subnet hay Public Subnet?

			Nhiều người chọn:
			Public

			Sai.

			Cần kiểm tra:
			Route Table
			Nếu không có:

			0.0.0.0/0 → IGW
			thì vẫn là Private Subnet.

# 6. Route Table liên quan thế nào?

	Subnet không tự biết đường đi.
	Route Table là người chỉ đường.

	Ví dụ:
		Subnet
		10.0.1.0/24


	Gắn Route Table:

	Destination	Target
	0.0.0.0/0	IGW
	=> Public


	Nếu:

	Destination	Target
	0.0.0.0/0	NAT Gateway
	=> Private subnet có Internet Outbound



	AWS Trap số 3

	NAT Gateway đặt ở đâu?
	Đáp án:
		Public Subnet
		Không được đặt trong Private Subnet.

# 8. CIDR và Subnet 


	Ví dụ:

	VPC:

		10.0.0.0/16

	Chia:
		10.0.1.0/24
		10.0.2.0/24
		10.0.3.0/24
		10.0.4.0/24

	Mỗi subnet:
		256 IP

	Vì:
		2
		32−24
		=256

		Nhưng AWS giữ lại 5 IP đầu/cuối.
			AWS reserved:

			Network address
			Router
			DNS
			Future use
			Broadcast equivalent
			
			
			
	Nên dùng được:
		251 IP



# Tóm tắt 1 câu duy nhất để đi thi
Subnet là vùng mạng con trong VPC; Public hay Private không phụ thuộc Public IP mà phụ thuộc Route Table. Public Subnet có đường tới Internet Gateway, Private Subnet thì không. Một Subnet chỉ thuộc một AZ.
