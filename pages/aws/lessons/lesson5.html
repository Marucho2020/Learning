# Region là gì?

	Region là một khu vực địa lý lớn nơi nhà cung cấp cloud đặt hạ tầng của mình.
	
Ví dụ trong Amazon Web Services:

	AWS Asia Pacific (Tokyo) Region
	AWS Asia Pacific (Singapore) Region
	AWS US East (N. Virginia) Region
	
		Mỗi Region nằm ở một vị trí địa lý khác nhau trên thế giới.
	
		World
		├ Tokyo Region
		├ Singapore Region
		├ Frankfurt Region
		└ Virginia Region



#	2. Availability Zone là gì?

		Availability Zone (AZ) là các data center độc lập bên trong một Region.

	Ví dụ:
		Tokyo Region
		├ AZ-A
		├ AZ-B
		└ AZ-C

			tách biệt vật lý
			có điện riêng
			có mạng riêng
			nếu một AZ hỏng → AZ khác vẫn hoạt động



# 3. Quan hệ giữa Region và AZ

	## Cấu trúc thật của cloud:

		Region
		↓
		Availability Zone
		↓
		Data Centers
		↓
		Servers


	## Ví dụ:

Tokyo Region
   ├ AZ-A
   │   └ Data Center
   ├ AZ-B
   │   └ Data Center
   └ AZ-C
       └ Data Center
	   
	   
	   
	   
# 	4. So sánh Region vs Availability Zone   
	   
	   
| Tiêu chí    | Region                 | Availability Zone |
| ----------- | ---------------------- | ----------------- |
| Quy mô      | lớn                    | nhỏ hơn           |
| Vị trí      | khu vực địa lý         | data center       |
| Số lượng    | ít hơn                 | nhiều hơn         |
| Khoảng cách | xa (giữa các quốc gia) | gần nhau          |
| Mục đích    | triển khai toàn cầu    | high availability |
	   
	   
	   
	   
# 	  5. Ví dụ thực tế 
	   
		Nếu bạn làm app cho user ở Nhật:
	    Bạn sẽ chọn region:
			👉 AWS Asia Pacific (Tokyo) Region
	   
		để latency thấp.Sau đó deploy server ở nhiều AZ:
			
			Tokyo Region
			
			AZ-A
			└ Server
			
			AZ-B
			└ Server	   
		Nếu AZ-A chết → AZ-B vẫn chạy.
	   


---------------- 日本語 -------------

# Regionとは

	**Region（リージョン）**とは、
	
	クラウドプロバイダーがインフラを設置している
	地理的な大きなエリアのことです。


例えば：

		AWS Asia Pacific (Tokyo) Region
		AWS Asia Pacific (Singapore) Region
		AWS US East (N. Virginia) Region

各Regionは 世界の異なる場所 にあります。




# Availability Zoneとは

**Availability Zone（AZ）**とは、
	1つのRegionの中にある物理的に分離されたデータセンター群です。

例：
	Tokyo Region
	
Tokyo Region
 ├ AZ-A
 ├ AZ-B
 └ AZ-C
 
	各AZは：
		- 電源が独立
		- ネットワークが独立
		- 障害が隔離されている
		
		
		
# RegionとAZの構造
		
	クラウドインフラの構造：	
		
Region
   ↓
Availability Zone
   ↓
Data Center
   ↓
Server		
		
		
| 項目 | Region  | Availability Zone |
| -- | ------- | ----------------- |
| 規模 | 大きい     | 小さい               |
| 位置 | 地理的エリア  | データセンター           |
| 数  | 少ない     | 多い                |
| 距離 | 遠い      | 近い                |
| 目的 | グローバル展開 | 高可用性              |
		
		

	日本のユーザー向けアプリの場合：
		👉 AWS Asia Pacific (Tokyo) Region
		
		を選びます。
		そして複数AZにサーバーを配置します。
		
		Tokyo Region
		
		AZ-A
		└ Server
		
		AZ-B
		└ Server
		
		AZ-Aが障害になっても
		AZ-Bがサービスを継続できます。



# AWS試験でよく出るポイント 

	覚えるべき定義： 
		Region:
			世界の特定の地理的エリア

		Availability Zone:
			Region内の物理的に分離されたデータセンター
