# Bản chất 

	Best Practice Cost nghĩa là:
		Thiết kế hệ thống sao cho chỉ trả tiền cho đúng lượng tài nguyên cần dùng, đúng thời điểm cần dùng, và luôn theo dõi để tối ưu liên tục.


	AWS không coi Cost Optimization là:
		"Làm mọi thứ rẻ nhất."

	AWS coi Cost Optimization là:
		Balance giữa Cost và Business Value.


	Ví dụ:
		Nếu một EC2 giá rẻ nhưng downtime khiến công ty mất hàng triệu đô doanh thu thì đó không phải tối ưu chi phí.


# 5 nguyên tắc Cost Optimization của AWS

| Best Practice                  | Ý nghĩa                                  |
| ------------------------------ | ---------------------------------------- |
| Right-size resources           | Chọn đúng kích thước tài nguyên          |
| Stop paying for idle resources | Không trả tiền cho tài nguyên không dùng |
| Use managed services           | Để AWS quản lý thay vì tự vận hành       |
| Measure everything             | Theo dõi Cost liên tục                   |
| Choose pricing model wisely    | Chọn mô hình thanh toán phù hợp          |


	Đây chính là tư duy xuyên suốt của Cost Optimization Pillar.


# 1. Right-size Resources

	Đây là Best Practice xuất hiện nhiều nhất.

	Ví dụ:

	Bạn chạy:
		EC2 m7i.4xlarge
		CPU chỉ dùng
		10%
		RAM dùng
		15%
		=> 85~90% tài nguyên bị lãng phí.
	AWS sẽ khuyên:
		↓
		m7i.large
		↓
		m7i.xlarge
		↓
		Hoặc Auto Scaling.



	Dịch vụ hỗ trợ:
		AWS Compute Optimizer
		Trusted Advisor
		CloudWatch Metrics

	Keyword:
		Right sizing
		Over-provisioned
		Underutilized


	Ví dụ đời thường
		Mua xe bus 45 chỗ.

	Ngày nào cũng chỉ chở:
		2 người.

	Chi phí vẫn như xe bus.
		Đó chính là over-provisioning.


2. Stop Paying for Idle Resources

	AWS tính tiền ngay cả khi:
		EC2 bật
		nhưng
		không ai dùng.

	Ví dụ:

	Developer:
		Bật EC2
		18:00 về nhà
		8:00 sáng hôm sau mới dùng.

	14 tiếng đó:

		vẫn tính tiền.
		Best Practice:
		Auto Stop
		Auto Start
		Lambda Scheduler
		Instance Scheduler


	Ví dụ khác:
		EBS Volume không attach
		Elastic IP không dùng
		NAT Gateway không có traffic
		Snapshot cũ
		ALB không dùng

	Đều tốn tiền.


# Ví dụ System Design 


		Dev Environment
			8:00
			↓
			Auto Start
			19:00
			↓
			Auto Stop

		Tiết kiệm:
			70~80%



# 3. Use Managed Services



Đây là điểm nhiều người không hiểu.

	AWS luôn thích:
		Managed Service
		hơn
		Self-managed.
		Ví dụ
	Thay vì:
		EC2
		↓
		MySQL
		↓
		Backup
		↓
		Monitoring
		↓
		Patch
		↓
		HA
		↓
		Scaling
		↓
		Ops Team



	AWS muốn:

	Amazon RDS

	Lý do:
		ít nhân sự
		ít downtime
		ít lỗi
		ít vận hành
	=> Tổng Cost thấp hơn.
		Không chỉ là giá EC2.

	Mà là:
		Total Cost of Ownership (TCO).



# TCO (Total Cost of Ownership)


	Đề thi rất thích.

	TCO gồm:
		Hardware
		Software
		Electricity
		Data Center
		Network
		Maintenance
		Human
		Security
		Monitoring


	AWS luôn tối ưu:
		TCO
		không chỉ:
		Monthly Bill.



# 4. Measure Everything

		Không đo
		↓
		Không tối ưu được.

	AWS có rất nhiều dịch vụ.

| Service                | Công dụng            |
| ---------------------- | -------------------- |
| Cost Explorer          | Phân tích Cost       |
| Budgets                | Cảnh báo             |
| Cost Anomaly Detection | Phát hiện bất thường |
| CUR                    | Báo cáo chi tiết     |
| Trusted Advisor        | Gợi ý tiết kiệm      |
| Compute Optimizer      | Right sizing         |

	Đây là nhóm dịch vụ Cost Management.

	Ví dụ

	Bạn thấy:
		EC2
		↓
		500 USD
		Nhưng:
		Không biết instance nào.
		Không biết team nào.
		Không biết project nào.
		=> Không tối ưu được.

	AWS khuyên:

		Tag
		↓
		Cost Allocation Tags
		↓
		Cost Categories



# 5. Chọn Pricing Model đúng
	AWS có rất nhiều model.


| Model         | Khi nào dùng         |
| ------------- | -------------------- |
| On-Demand     | Không biết workload  |
| Reserved      | Dùng lâu dài         |
| Savings Plans | Linh hoạt hơn RI     |
| Spot          | Chấp nhận bị thu hồi |
| Free Tier     | Học tập              |


	Ví dụ
		Web server
		24/24
		365 ngày
		Nếu vẫn dùng
		On-Demand
		↓
		Lãng phí.
	Nên dùng:
		Reserved
		hoặc
		Savings Plans.


# Mapping với dịch vụ AWS 

| Mục tiêu             | AWS Service                               |
| -------------------- | ----------------------------------------- |
| Theo dõi chi phí     | Cost Explorer                             |
| Cảnh báo ngân sách   | AWS Budgets                               |
| Phát hiện bất thường | Cost Anomaly Detection                    |
| Gợi ý tối ưu         | Trusted Advisor                           |
| Right sizing         | Compute Optimizer                         |
| Auto Scale           | Auto Scaling                              |
| Tự động tắt EC2      | EventBridge + Lambda / Instance Scheduler |
| Phân bổ chi phí      | Cost Allocation Tags                      |
| Báo cáo chi tiết     | Cost & Usage Report (CUR)                 |



# Trade-off


| Lựa chọn           | Ưu điểm                        | Nhược điểm                           |
| ------------------ | ------------------------------ | ------------------------------------ |
| On-Demand          | Linh hoạt                      | Đắt nhất nếu chạy liên tục           |
| Reserved Instances | Tiết kiệm lớn                  | Cam kết thời hạn                     |
| Savings Plans      | Linh hoạt hơn RI               | Vẫn cần cam kết mức chi tiêu         |
| Spot               | Rẻ nhất (có thể giảm tới ~90%) | Có thể bị AWS thu hồi bất cứ lúc nào |
| Managed Service    | Giảm chi phí vận hành          | Ít quyền kiểm soát hơn               |
