# 1. Bản chất của AWS Pricing Model 

	Nói ngắn gọn:
		AWS không bán server. AWS bán "mức độ sử dụng tài nguyên".

	Đây là tư duy quan trọng nhất.

	Trong On-premise:
		Mua server
		Mua storage
		Mua network
		Trả tiền trước
		Dùng hay không vẫn mất tiền.

	AWS thì khác.


		AWS chia tài nguyên thành từng đơn vị rất nhỏ.

		Ví dụ:
			1 giờ EC2
			1 GB lưu S3
			1 triệu request Lambda
			1 GB dữ liệu truyền Internet
			1 triệu token Bedrock

		Bạn dùng bao nhiêu
		→ trả bấy nhiêu.


		Đó gọi là
			Pay as you go
		Đây là triết lý gần như toàn bộ Cloud của AWS.


# 2. Có những Pricing Model nào?

| Pricing Model           | Bản chất                         |
| ----------------------- | -------------------------------- |
| Pay-as-you-go           | Trả theo lượng sử dụng           |
| Savings Plans           | Cam kết sử dụng để được giảm giá |
| Reserved Instances (RI) | Đặt trước EC2/RDS để giảm giá    |
| Spot Instances          | Mua tài nguyên dư thừa rất rẻ    |
| Free Tier               | Miễn phí giới hạn                |


# 3. Pay-as-you-go
	Đây là mô hình mặc định.

		AWS Mapping
			EC2
			Lambda
			S3
			DynamoDB
			Bedrock
			API Gateway

		đều dùng Pay-as-you-go.

		Từ khóa đề thi
			Pay only for what you use
			No upfront cost
			Elastic
			Consumption based pricing

			Chỉ cần thấy mấy chữ này
		=> Pay-as-you-go.



# 4. Reserved Instance (RI) 


	Bản chất

	AWS hỏi bạn:
		"Anh chắc chắn sẽ dùng server này lâu dài không?"

	Nếu

	Có
		AWS giảm giá.



# 5. Savings Plans


	Đây là phần nhiều người nhầm nhất.

	Reserved Instance
		Cam kết
		Instance

	Savings Plans
		Cam kết
		Số tiền sử dụng mỗi giờ



		Bạn cam kết
			10 USD/giờ.

		AWS không quan tâm

			bạn chạy
			EC2 nào.

		Miễn tổng tiền
			≈10 USD/giờ.

		Bạn vẫn được giảm giá.


		Ví dụ

		Hôm nay
			m5.large

		Mai
			c6i.large

		Mốt
			m7g.large

		vẫn được giảm.



	Trade-off
		Linh hoạt hơn RI
		Nhưng vẫn phải cam kết
		1 hoặc 3 năm.



	Từ khóa
		Flexible
		Commit spending
		Compute Savings Plan



	6. Spot Instance


		Bản chất

		AWS Data Center luôn có server rảnh.
			AWS bán rẻ.
			Có thể
			giảm
			90%.
			Nhưng
			AWS có quyền lấy lại.
			Không báo trước nhiều.

			Ví dụ
				AWS cần tài nguyên.
				EC2 Spot
				bị terminate.



		Không dùng Spot cho
			Database
			Payment
			Banking
			ERP



		Dùng Spot cho
			AI Training
			Batch
			Big Data
			Render
			Video Encoding
			Simulation


		Từ khóa
			interruptible
			spare capacity
			up to 90%
