# 1. Bản chất của EBS (Định nghĩa không mơ hồ)

	Amazon EBS (Elastic Block Store) là ổ cứng mạng (Network Block Storage) được AWS tạo ra để gắn (Attach) vào EC2.


	Nói dễ hiểu:

		EC2 là máy tính.
		EBS là ổ SSD/HDD của máy tính đó.

		Nhưng ổ cứng này không nằm trong máy, mà nằm trong hệ thống lưu trữ riêng của AWS và được kết nối qua mạng tốc độ cao.


	Đây là điểm rất quan trọng.
		Nhiều người mới học tưởng EBS nằm trong EC2.
		Không.


          AWS Datacenter

      +----------------------+
      |     EC2 Instance     |
      |      CPU / RAM       |
      +----------+-----------+
                 |
      Network Storage
                 |
      +----------v-----------+
      |        EBS           |
      | SSD / HDD Volume     |
      +----------------------+

		CPU chết không có nghĩa dữ liệu mất.
		Bạn chỉ cần attach EBS sang EC2 khác.
		Đây chính là bản chất của cloud.
		

# 2. EBS lưu cái gì?


	Thông thường:
		Operating System (Windows/Linux)
		Application
		Database
		Log
		User Data


	Ví dụ:

	EC2
		Ubuntu

	/
		├── nginx
		├── mysql
		├── app.jar
		├── logs
		└── home

		Toàn bộ đều nằm trên EBS.


# 3. Block Storage là gì?


| Loại           | AWS |
| -------------- | --- |
| Block Storage  | EBS |
| File Storage   | EFS |
| Object Storage | S3  |


	Block Storage nghĩa là:
		Hệ điều hành nhìn thấy giống hệt một ổ SSD.

	Ví dụ Linux:


	/dev/xvda
	/dev/nvme0n1

	Bạn format
		mkfs.ext4

	mount
		mount /dev/xvdf /data


	AWS không quan tâm bên trong.
	EC2 tự quản lý filesystem.

# 4. Vì sao AWS tách EBS khỏi EC2?

	Đây là tư duy Cloud.
	Nếu SSD nằm trong EC2:	
	
		EC2 chết
		↓
		SSD chết
		↓
		Data mất

	Nếu SSD tách riêng

		EC2 chết
		↓
		Detach EBS
		↓
		Attach EC2 mới
		↓
		Chạy tiếp

	Đây chính là High Availability ở mức Compute.

# 	9. Snapshot là gì?
	
	Snapshot là Backup của EBS.
	Không phải copy nguyên ổ.
	AWS chỉ lưu block đã sử dụng.	
		
		
	
	Ví dụ
		Volume
		100GB
		Nhưng dùng
		25GB
		Snapshot chỉ khoảng
		25GB
		
	và lưu trên S3 (người dùng không truy cập trực tiếp).

# 	12. Các loại EBS	
		
		
| Type    | Dùng cho                     |
| ------- | ---------------------------- |
| gp3     | Mặc định, đa số workload     |
| gp2     | Cũ hơn gp3                   |
| io1/io2 | Database cần IOPS rất cao    |
| st1     | HDD throughput               |
| sc1     | Archive, dữ liệu truy cập ít |
		
		
		
		
# 	Trade-off	
		
		
| gp3     | io2                |
| ------- | ------------------ |
| Rẻ      | Đắt                |
| Nhanh   | Rất nhanh          |
| Web App | Oracle, SQL Server |
		
		
		
# 	13. So sánh EBS vs EFS vs S3	
		
|             | EBS              | EFS          | S3                   |
| ----------- | ---------------- | ------------ | -------------------- |
| Storage     | Block            | File         | Object               |
| Mount       | 1 EC2 (mặc định) | Nhiều EC2    | Không mount như disk |
| OS thấy như | SSD              | Folder       | Bucket               |
| AZ          | Một AZ           | Multi-AZ     | Multi-AZ             |
| Use case    | OS, DB           | Shared Files | Backup, Image, Video |

# 	18. Những bẫy AWS rất thích gài	
		
| Bẫy                               | Thực tế                                            |
| --------------------------------- | -------------------------------------------------- |
| EBS dùng chung cho nhiều EC2      | Sai (mặc định chỉ attach một EC2)                  |

| Database luôn dùng gp3            | Không hẳn, workload IOPS cao nên dùng io2          |
		
		


# 19. Cách nhớ cực lâu

	Hãy tưởng tượng AWS là một trung tâm dữ liệu:
		EC2 = Bộ não (CPU + RAM): xử lý tính toán, có thể thay thế nhanh.
		EBS = Ổ SSD gắn qua dây cáp mạng tốc độ cao: lưu hệ điều hành và dữ liệu bền vững.
		Snapshot = Chụp ảnh trạng thái của ổ SSD, cất vào kho lưu trữ (S3) để khôi phục khi cần.

	Công thức nhớ:
		EC2 xử lý → EBS lưu trữ → Snapshot sao lưu.

	Nếu gặp câu hỏi có các từ khóa như OS, boot, database disk, persistent block storage, gần như bạn nên nghĩ ngay đến EBS.
