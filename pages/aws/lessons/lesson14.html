# IaaS là gì? (Infrastructure as a Service) 

	IaaS = bạn thuê hạ tầng, còn lại tự lo
	
		AWS cho bạn: VM, network, storage
		Bạn tự quản:

			- OS
			- runtime
			- app
			- security config (phần lớn)

	👉 Nói thẳng:
	Bạn vẫn là DevOps, AWS chỉ là “thợ điện + phòng server”
	giống thuê vps 


# PaaS là gì?(Platform as a Service)

	PaaS = AWS lo OS + runtime, bạn chỉ code
		Bạn chỉ deploy code
	
		AWS lo:
		
			OS
			patch
			scaling (một phần)
	
	👉 Nói thẳng:
	Bạn tập trung logic, không đụng infra	
	Giống Vercel




#	SaaS (Software as a Service)

	SaaS = dùng luôn, không quản gì

		- Mở ra là dùng
		- Không deploy, không config infra

	👉 Nói thẳng:
	Bạn là user, không phải engineer
	



# II. SO SÁNH GỐC RỄ (CÁI QUAN TRỌNG NHẤT)


| Layer    | IaaS | PaaS | SaaS |
| -------- | ---- | ---- | ---- |
| App code | bạn  | bạn  | ❌    |
| Runtime  | bạn  | AWS  | ❌    |
| OS       | bạn  | AWS  | ❌    |
| Infra    | AWS  | AWS  | AWS  |



	👉 Insight cực quan trọng:
	Càng lên cao → càng ít control → càng ít trách nhiệm


# III. MAPPING AWS

		## IaaS
				- EC2
				- EBS
				- VPC
			👉 Bạn SSH vào server → IaaS


	
		## PaaS
		
			- Elastic Beanstalk
			- AWS Lambda (serverless)
			- RDS (semi-managed)	
			
			👉 Bạn không quản OS → PaaS
		
		
			
		## SaaS
			- Gmail
			- Slack
			- AWS QuickSight		
				👉 Không deploy gì → SaaS
				
				
				
				
# IV. VÍ DỤ THỰC TẾ 			
				
	Case 1: Startup nhỏ, cần control cao	
				
		👉 chọn IaaS
		
		EC2 + Docker
		tự config everything
		
		👉 lý do:
		
		linh hoạt
		nhưng tốn công				
				
				
				
	Case 2: Dev muốn deploy nhanh

		👉 chọn PaaS
		
		code → push → chạy
		
		👉 ví dụ:
		
		Elastic Beanstalk auto scale



	Case 3: Công ty chỉ cần tool

		👉 chọn SaaS
		
		dùng CRM, email
		
		👉 không cần dev




VI. BẪY AWS HAY GÀI

	## Bẫy 1: RDS là gì?
		👉 nhiều người nghĩ SaaS → SAI
		
		RDS = PaaS (managed DB)
		

	## Bẫy 2: Lambda là gì?

		👉 Serverless ≠ SaaS
		
		Lambda = PaaS Bạn vẫn viết code
	
				
				
	## 	Bẫy 3: EC2 auto scaling → có phải PaaS?		
				
		👉 KHÔNG
		
		vẫn phải quản OS → vẫn là IaaS				
						
				
	## 	Bẫy 4: “Managed” ≠ SaaS		
				
		Managed DB = PaaS
		SaaS = không deploy gì				
				
				
VIII. CÁCH AWS RA ĐỀ				
				
	AWS không hỏi:
		“IaaS là gì?”
	
	Nó hỏi:
		“Bạn muốn control tới đâu?”				
			

	👉 Rule:
		Muốn control sâu → IaaS
		Muốn dev nhanh → PaaS
		Không muốn dev → SaaS


	👉 Trick:
	
		Có “manage OS” → IaaS
		Có “no need OS” → PaaS
		Có “end-user tool” → SaaS



IX. TRADE-OFF(THỨ AWS MUỐN BẠN HIỂU)


|             | IaaS | PaaS  | SaaS     |
| ----------- | ---- | ----- | -------- |
| Control     | cao  | trung | thấp     |
| Effort      | cao  | thấp  | rất thấp |
| Flexibility | cao  | trung | thấp     |


	👉 Nhận định thẳng:
		Junior hay chọn SaaS/PaaS
		Senior phải biết khi nào cần IaaS

# IaaSとは？

	インフラだけ提供、残りは自分で管理

		- EC2など
		- OS・アプリは自分で管理
	👉 フルコントロール



# PaaSとは？

	OSやランタイムはAWSが管理
		開発者はコードだけ

	👉 インフラ管理不要



# SaaSとは？
	すぐ使えるサービス
	設定・デプロイ不要
	👉 エンドユーザー向け


			
# II. 比較 

| 項目     | IaaS | PaaS | SaaS |
| ------ | ---- | ---- | ---- |
| コントロール | 高    | 中    | 低    |
| 管理負担   | 高    | 低    | ほぼなし |




# III. AWSサービス対応

	- IaaS：EC2
	- PaaS：Elastic Beanstalk / Lambda / RDS
	- SaaS：QuickSight


# IV. 試験キーワード


	## IaaS 
		- full control
		- manage OS		

	## PaaS
		- deploy application
		- no OS management

	## SaaS
		- ready to use
		- no setup


# V. トラップ

	- RDS = PaaS（SaaSではない）
	- Lambda = PaaS
	- Auto Scaling EC2 = IaaS


# VI. 出題パターン

	AWSはこう聞く：
		「どこまで管理したい？」

			
			
# 	VII. 結論		
			
	- control重視 → IaaS
	- 開発スピード → PaaS
	- 利用のみ → SaaS
