 # I. Giải thích bằng tiếng Việt 
 
 
 1. NAT Gateway là gì? (Bản chất thật sự)

	NAT Gateway tồn tại để:
		Cho phép tài nguyên trong Private Subnet đi ra Internet nhưng không cho Internet đi vào bên trong.
	 
 
	 Nói đơn giản:
		EC2 trong Private Subnet cần:
		download package
		gọi API bên ngoài
		truy cập S3 Public Endpoint
		cập nhật Windows/Linux

	=> cần ra Internet.
	 
	 Nhưng:
		Không muốn gán Public IP
		Không muốn Internet truy cập trực tiếp vào EC2

	=> NAT Gateway giải quyết bài toán này.

#  3. NAT Gateway hoạt động như thế nào?



Bước 1

	Private EC2 gửi request:
		8.8.8.8

	Ví dụ:
		10.0.2.10

	gửi request tới:
		api.openai.com




Bước 2

	Route Table của Private Subnet:
		0.0.0.0/0
			->
		NAT Gateway
		Traffic được gửi sang NAT Gateway.


	 
	 
	 
Bước 3

	NAT Gateway thay địa chỉ nguồn

	Trước:
		Source:
		10.0.2.10

	Sau:
		Source:
		54.x.x.x

	(EIP của NAT Gateway)
		Đây chính là:
		Network Address Translation

		(NAT)	 
	 
	 
	 
Bước 4

	Internet trả lời
		OpenAI API
			↓
		54.x.x.x
		NAT Gateway nhận response.	 
		 
	 
Bước 5

	NAT Gateway gửi lại
		54.x.x.x
			↓
		10.0.2.10
		Private EC2 nhận được kết quả.	 
 
 
 
 
 
 
# Bản chất sâu hơn   
 
 
 
Internet KHÔNG biết:

	10.0.2.10 tồn tại.

	Internet chỉ biết:
		54.x.x.x
		(NAT Gateway)

	NAT Gateway giống:
		Lễ tân của một công ty.

	Nhân viên:
		Private EC2

	Khách hàng:
		Internet

	Khách hàng chỉ nói chuyện với lễ tân.
	Không ai biết bên trong công ty có ai.
	 
	 
 
 
 
#  4. NAT Gateway phải nằm ở đâu?
 
 
 
Câu thi cực kỳ hay hỏi.

Đáp án:
	NAT Gateway PHẢI nằm trong Public Subnet.
	Vì NAT Gateway phải truy cập Internet. 
	 
	 
 
 
Kiến trúc đúng:  
 
 
Internet
    |
IGW
    |
Public Subnet
    |
NAT Gateway
    |
Private Subnet
    |
EC2 
 
 
 
# 5. Route Table cấu hình thế nào?  
 
	 Public Subnet
			0.0.0.0/0
				->
			Internet Gateway
		 
 
 
	Private Subnet
		0.0.0.0/0
			->
		NAT Gateway 
		 
	 
#  6. NAT Gateway cần gì để hoạt động?

 
	Điều kiện 1
		Public Subnet

	Điều kiện 2
		Internet Gateway

	Điều kiện 3
		Elastic IP

	NAT Gateway luôn gắn:

		Elastic IP
		để Internet biết trả lời về đâu.
