# 🔥 ETL là gì (nói thẳng, ngắn gọn) 
	ETL = Extract → Transform → Load


	👉 Hiểu đúng bản chất:
		Extract: lấy dữ liệu từ nhiều nguồn (DB, log, API…)
		Transform: xử lý dữ liệu (clean, join, tính toán…)
		Load: đẩy vào nơi dùng (data warehouse, S3, BI…)


# ⚡ Ví dụ thực tế

	Một công ty:

		- Lấy dữ liệu user từ MySQL
		- Lấy log từ server
		- Chuẩn hóa format (timestamp, currency…)
		- Đưa vào hệ thống phân tích

	👉 Đó chính là ETL pipeline



# 🧠 Mapping AWS

		Extract → DynamoDB / RDS / S3
		Transform → AWS Glue / Lambda / EMR
		Load → S3 / Redshift / Athena


# 🎯 Chốt 1 câu 
	ETL = “gom dữ liệu → xử lý → đưa vào hệ phân tích”


-------------- 🇯🇵 日本語版-------------

# 🔥 ETLとは

	ETL = Extract → Transform → Load

		Extract：データ取得
		Transform：データ加工
		Load：保存・分析基盤へ投入



# ⚡ 具体例 

		DBやログからデータ取得
		フォーマット変換・クレンジング
		分析用システムへ格納

		👉 これがETLパイプライン

# 🧠 AWS対応 

	Extract → RDS / DynamoDB / S3
	Transform → AWS Glue / Lambda / EMR
	Load → S3 / Redshift / Athena

🎯 まとめ
	ETL = 「データ収集 → 加工 → 分析基盤へ投入」
