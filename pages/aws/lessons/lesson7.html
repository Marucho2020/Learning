# 1. Scalability

	## Khái niệm 
		Scalability (khả năng mở rộng) là khả năng của hệ thống tăng hoặc 
		giảm năng lực xử lý khi nhu cầu tăng lên.

		###Điểm quan trọng:

		- Hệ thống có thể mở rộng để xử lý nhiều request hơn
		- Thường không cần thay đổi kiến trúc
		- Việc mở rộng không nhất thiết phải tự động

		### Ví dụ thực tế
			Một website bán hàng.
				Ban đầu có:1 server xử lý 1000 user.
				
			Sau đó traffic tăng lên.
			Ta mở rộng bằng cách:
				nâng cấp server mạnh hơn hoặc
				thêm nhiều server mới
		
		-> Đây chính là Scalability.
		
		

	## Hai loại Scalability 
	
	
		### 1️ Vertical Scaling (Scale Up) 
				Tăng sức mạnh của một máy
	
			Ví dụ: RAM 8GB → 32GB
				   CPU 2 core → 16 core

				nâng cấp instance của Amazon EC2 : t3.micro → t3.large


			Nhược điểm:

				có giới hạn vật lý
				thường phải restart
				
				
				
				
				
				
		### 2️ Horizontal Scaling (Scale Out)	
				
				
			Trong AWS thường dùng:
			
				- Amazon EC2
				- Elastic Load Balancing
				- AWS Auto Scaling
			
			Load Balancer sẽ chia request				
							
				
				
				
				
			Ví dụ dễ hiểu	
			Scalability là:	
				tăng thêm bàn
				thuê thêm đầu bếp			
				
			Nhưng không nhất thiết phải làm ngay lập tức.	
				
				
				
				
------------------	日本語	----------------

	**Scalability（スケーラビリティ）**とは、
		システムの負荷が増えたときに処理能力を拡張できる能力				
		のことです。		
				
				
# 重要なポイント：

		- ユーザーやトラフィックが増えても対応できる
		- システムを拡張できる
		- 自動である必要はない

	
				
				
#	スケーリングの2種類		
				
	## ① Vertical Scaling（スケールアップ） 
		1台のサーバーの性能を上げる方法。
	
			CPUを増やす
			メモリを増やす				
		AWSでは	のインスタンスサイズを変更します。	
				
	## 欠点：			
		- 上限がある
		- 再起動が必要な場合がある		



# ② Horizontal Scaling（スケールアウト）

	サーバーの数を増やす方法。
		1台 → 10台

	## AWSでは： 

		- Amazon EC2
		- Elastic Load Balancing
		- AWS Auto Scaling
		を使ってトラフィックを分散します。
