# 1. Load Balancing là gì

	Load Balancing = phân phối request đến nhiều server
	Không để 1 server bị quá tải trong khi server khác rảnh.



	## Ví dụ đơn giản 

	Bạn có 3 server:
		A   B   C

	User gửi request:
			Nếu không có Load Balancer: → tất cả dồn vào A → chết


		Có Load Balancer:
			→ chia đều A / B / C


	## AWS dùng gì

	Elastic Load Balancing (ELB)



	## Vai trò thực tế

	Load Balancer đứng giữa:
		User → Load Balancer → Servers


	## Lợi ích

	tăng performance
	tránh quá tải
	hỗ trợ High Availability
	tự động route request




# 2. Cách Load Balancer hoạt động

	Nó quyết định: request này đi server nào


	## Các thuật toán phổ biến 

		### 1.Round Robin → chia đều
			A → B → C → A → B → C



		### 2. Least Connections
			→ server nào ít request nhất → nhận



		### 3. IP Hash 
			→ user A luôn vào server A
			
			👉 Trong AWS bạn không cần config sâu như Nginx
			→ AWS xử lý sẵn.




# 3. Các loại Load Balancer trong AWS


	## 1️⃣ Application Load Balancer (ALB)
		Dùng khi:
			HTTP / HTTPS



		Đặc điểm
			- hiểu request (URL, path, header)
			- routing thông minh


		Ví dụ
			/api → server A
			/image → server B





	## 2️⃣ Network Load Balancer (NLB)

		Dùng khi:
			- TCP / UDP
			- cần tốc độ cao


		Đặc điểm
			rất nhanh (low latency)
			không hiểu HTTP




	3️⃣ Gateway Load Balancer (GWLB)

		Dùng khi:

			- security appliance
			- firewall, IDS





# 4. Load Balancing + Auto Scaling (combo hay ra đề)

	## Khi traffic tăng:
		- Auto Scaling → thêm server
		- Load Balancer → phân phối

	
	## Flow chuẩn AWS
	
		User
		↓
		Load Balancer
		↓
		Auto Scaling Group
		↓
		EC2 instances
		
		
		
		
	## 	Service liên quan
		- AWS Auto Scaling
		- Amazon EC2		
		
		
	
		
		
# 	5. Health Check 
		
	Load Balancer sẽ:
	
		check server còn sống không
		Nếu chết:
			→ không gửi request vào

		Ví dụ
			- Server A (OK)
			- Server B (DOWN)

		→ chỉ gửi vào A



# đề AWS


	route request based on URL
		=> Application Load Balancer


	ultra-low latency, TCP traffic
		=> Network Load Balancer


	distribute traffic across instances
		=> Load Balancer

	
		
		


--------------日本語 -----------

**Load Balancing（ロードバランシング）**とは：
		複数のサーバーにリクエストを分散する仕組み




# イメージ
	ユーザー → Load Balancer → サーバー群



# 主なアルゴリズム

	Round Robin
	Least Connections
	IP Hash
	
	※ AWSでは基本的に自動管理



	## 1️⃣ Application Load Balancer (ALB)
	
		HTTP/HTTPS専用
		URLやパスでルーティング可能




	## 2️⃣ Network Load Balancer (NLB)

		- 高速（低レイテンシ）
		- TCP/UDP対応




	## 3️⃣ Gateway Load Balancer (GWLB) 
		セキュリティ機器向け


# 4. Load Balancing + Auto Scaling


	## トラフィック増加時：

		Auto Scaling → インスタンス追加
		Load Balancer → 負荷分散	





# 	5. Health Check 

	ヘルスチェック機能：
		正常なサーバーのみトラフィック送信
