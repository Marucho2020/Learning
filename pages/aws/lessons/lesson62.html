# 1. Amazon CloudFront là gì?
	CloudFront là dịch vụ CDN (Content Delivery Network) của AWS.

	Nói thẳng:
		CloudFront = hệ thống cache toàn cầu của AWS.
	Nó copy dữ liệu từ Origin (S3, ALB, EC2...) ra hàng trăm điểm trên thế giới (Edge Location).



	Khi user truy cập:
		Không phải lúc nào cũng về server gốc
		CloudFront sẽ trả dữ liệu từ cache gần user nhất

	=> Giảm latency
	=> Giảm tải Origin
	=> Giảm chi phí


# Ví dụ đời thường 

	Giả sử:
		Server đặt tại Tokyo
		User ở Paris

	Không dùng CloudFront:


	Paris
		↓
		Tokyo Server
		↓
	Paris
	Mỗi request phải đi nửa vòng trái đất.




# Dùng CloudFront:


	Tokyo Server
		  ↓
	Paris Edge Location (cache)

	User Paris
		  ↓
	Paris Edge

	Request không cần quay về Tokyo.

# CloudFront hoạt động thế nào?


	Lần đầu:
		User
		 ↓
		CloudFront
		 ↓
		Origin
		 ↓
		CloudFront Cache
		 ↓
		User



	Lần sau:
		User
		 ↓
		CloudFront Cache
		 ↓
		User

		Không cần Origin.


	Từ khóa đề thi
		Edge Location
		Cache
		TTL
		Global Distribution
		Low Latency
		Static Content
		Dynamic Content
		Content Delivery
	Nhìn thấy các từ này thường nghĩ đến:

	👉 CloudFront


# CloudFront giúp tiết kiệm tiền như thế nào?


	Đây là điểm AWS rất thích hỏi.

	Giả sử:
		100 triệu lượt tải ảnh

	Không dùng CloudFront:
		100 triệu request vào S3


	Dùng CloudFront:
		95 triệu request
		↓
		CloudFront Cache

		5 triệu request
		↓
		S3
		S3 ít request hơn.
	=> Chi phí giảm.



# Các loại Cache


	## Cache Hit
		CloudFront đã có dữ liệu.
			User
			 ↓
			CloudFront
			 ↓
			Response
		Không gọi Origin.




	## Cache Miss
		CloudFront chưa có dữ liệu.
		User
		 ↓
		CloudFront
		 ↓
		Origin
		 ↓
		CloudFront Cache
		 ↓
		User	
			
	
	Đề thi cực thích hỏi:
		Muốn giảm chi phí và giảm latency?
	Đáp án:
	👉 Tăng Cache Hit Ratio

| Cache           | Không | Có         |
