# Edge Location là gì?

	Edge Location là máy chủ đặt gần người dùng nhất để phân phối nội dung nhanh hơn.
	Nó thường dùng trong CDN (Content Delivery Network).
		Trong Amazon Web Services,
		Edge Location chủ yếu được dùng bởi dịch vụ:

	👉 Amazon CloudFront




# Vấn đề nếu không có Edge Location

	Giả sử server của bạn nằm ở:
		👉 AWS Asia Pacific (Tokyo) Region

	Nhưng user ở:
		- Brazil
		- Europe
		- Mỹ

	Request sẽ phải đi:
		User (Brazil)
			↓
		Internet
			↓
		Tokyo Server	
		
		Khoảng cách rất xa → latency cao.
	
		Kết quả:
			- website chậm
			- video load chậm
			- game lag	
	
	
# 	Edge Location giải quyết vấn đề này
	
	## Edge Location là server cache đặt khắp thế giới.
	Ví dụ:
	
		User (Brazil)
			↓
		Edge Location (Brazil)
			↓
		Tokyo Server	
	
	## Quá trình hoạt động:
	
		1️ User request nội dung
		2️ Edge Location kiểm tra cache	
		
			Nếu đã có sẵn → trả về ngay
			
			Nếu chưa có  
						→ lấy từ origin server (Tokyo)
						→ lưu vào cache		
	
	## Kiến trúc thật 
	
		User
		↓
		Edge Location
		↓
		CloudFront
		↓
		Origin Server (EC2 / S3)	
	
				Origin server thường là:
				Amazon EC2
				Amazon S3	
	

# So sánh Region vs AZ vs Edge Location 

| Thành phần        | Vai trò                  |
| ----------------- | ------------------------ |
| Region            | khu vực cloud lớn        |
| Availability Zone | data center trong region |
| Edge Location     | server cache gần user    |



	## Cấu trúc: 

		Region
		↓
		Availability Zone
		↓
		Servers
		
		Edge Location
		↓
		near users


# Ví dụ thực tế  
	
Website video.
Server ở Tokyo.
User ở Mỹ.

	## Không có CDN:

		User (US)
		↓
		Tokyo server


	## Có CDN:
		Sau request đầu tiên:
		Edge Location sẽ cache video.
		User tiếp theo → load cực nhanh.

# Edge Locationとは

	ユーザーに近い場所に配置されたコンテンツ配信用サーバーです。
	主に CDN（Content Delivery Network） で使われます。
	AWSではこの仕組みを提供するサービスが：Amazon CloudFront です。



# なぜEdge Locationが必要か

	もしサーバーが👉 AWS Asia Pacific (Tokyo) Regionにあり、ユーザーが
		- ヨーロッパ
		- アメリカ
		- 南米
	にいる場合：

		User (US)
		↓
		Internet
		↓
		Tokyo Server

	距離が遠い → レイテンシーが高い。



# Edge Locationの仕組み

	Edge Locationは
	ユーザーに近い場所でコンテンツをキャッシュします。

	## 例：
		User
		↓
		Edge Location
		↓
		Tokyo Server



	## 処理の流れ：
		1️ ユーザーがリクエスト
		2️ Edge Locationがキャッシュ確認

	キャッシュあり → 即レスポンス
		Origin serverから取得 → キャッシュ保存



# 構成例 

	User
	↓
	Edge Location
	↓
	CloudFront
	↓
	Origin Server


	Origin serverは通常：
		Amazon EC2
		Amazon S3

	などです。


# Region / AZ / Edge Location


| 要素                | 役割               |
| ----------------- | ---------------- |
| Region            | 地理的クラウドエリア       |
| Availability Zone | Region内のデータセンター  |
| Edge Location     | ユーザー近くのキャッシュサーバー |
