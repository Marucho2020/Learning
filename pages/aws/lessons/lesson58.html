# 1. Định nghĩa bản chất (nói thẳng)
	Internet Gateway (IGW) là cổng kết nối giữa VPC và Internet.

	Nếu không có IGW:
		EC2 trong VPC không thể ra Internet
		Internet không thể vào EC2

	IGW không phải Router.
	IGW không phải NAT.
	IGW không phải Firewall.
	IGW chỉ là:
		"Cánh cổng giúp Route từ VPC đi ra Internet và từ Internet đi vào VPC."




	## Hình dung đời thường

	Nhà bạn nằm trong khu dân cư kín.
		VPC = Khu dân cư
		EC2 = Nhà trong khu dân cư
		Route Table = Biển chỉ đường
		Internet Gateway = Cổng ra đường quốc lộ


	Không có cổng:
		➡ Không ai ra ngoài được.
		➡ Không ai đi vào được.

	Có cổng nhưng không có biển chỉ đường:
		➡ Vẫn không đi được.

	Đây là điểm AWS rất thích ra đề.


# 2. Điều kiện để EC2 truy cập Internet

Nhiều người nghĩ:

	Có Internet Gateway là đủ.
	
	AWS yêu cầu đủ 3 điều kiện.


| Điều kiện                    | Bắt buộc |
| ---------------------------- | -------- |
| IGW attached vào VPC         | ✅        |
| Route Table có route tới IGW | ✅        |
| EC2 có Public IP             | ✅        |

	Thiếu 1 cái là không ra Internet được.


# Ví dụ
	VPC		10.0.0.0/16
		
	Public Subnet 	10.0.1.0/24 

	Route Table
		10.0.0.0/16 -> local
		0.0.0.0/0 -> igw-123456

	EC2
		Private IP:
		10.0.1.10

		Public IP:
		54.x.x.x
	=> Truy cập Internet được.



# 3. Route Table mới là thứ quyết định Public hay Private Subnet

	Đây là điểm cực kỳ quan trọng.
	AWS không hề có resource tên:
		Public Subnet
		Private Subnet

	AWS chỉ có:
		Subnet
		Public hay Private là do Route Table.
	

# Public Subnet 
	0.0.0.0/0 -> IGW
	Có đường ra Internet.


# Private Subnet 
	0.0.0.0/0 -> NAT Gateway

	Không có route trực tiếp tới IGW.
	Internet không thể initiate connection vào.
	
|                             | Public Subnet | Private Subnet |
| --------------------------- | ------------- | -------------- |
| Route tới IGW               | ✅             | ❌              |
| Route tới NAT               | Có thể        | ✅              |
| EC2 có Public IP            | Thường có     | Không          |
| Internet truy cập trực tiếp | ✅             | ❌              |

# 5. Internet Gateway có làm NAT không?
	Đây là câu hỏi senior hay hỏi.


	Câu trả lời:
		Có nhưng không phải NAT Gateway.
		IGW thực hiện:
		1-to-1 NAT

		Ví dụ:

		54.1.1.1
		↔
		10.0.1.10

		AWS gọi là:
		Public IP Translation


	Trong khi NAT Gateway:
		10.0.1.10
		10.0.1.11
		10.0.1.12

		↓
		54.1.1.1

		Nhiều Private IP dùng chung 1 Public IP.
		Đây là bản chất khác nhau.



# 6. Mapping với các dịch vụ AWS 

| Service               | Quan hệ với IGW                                 |
| --------------------- | ----------------------------------------------- |
| VPC                   | IGW attach vào VPC                              |
| Route Table           | Route tới IGW                                   |
| Public Subnet         | Sử dụng IGW                                     |
| NAT Gateway           | NAT Gateway phải nằm trong Public Subnet có IGW |
| ALB                   | Thường đặt trong Public Subnet có IGW           |
| Bastion Host          | Phải nằm trong Public Subnet có IGW             |
| EC2 Public Web Server | Cần IGW                                         |
