# CDN là gì? 

Nói ngắn gọn:
	CDN là hệ thống đặt nhiều bản sao dữ liệu ở nhiều nơi trên thế giới để đưa nội dung đến người dùng từ vị trí gần nhất.


	CDN không phải là nơi tạo dữ liệu.
	CDN không phải database.
	CDN không thay thế server gốc.
	CDN chỉ là:
		"Bộ nhớ đệm (cache) phân tán toàn cầu."


# Vì sao CDN tồn tại? 

	Internet có một sự thật:
	Khoảng cách vật lý càng xa ⇒ độ trễ (latency) càng cao.
	Ví dụ:

	Server đặt tại Tokyo.
	Người dùng tại:
		New York
		Paris
		London

	Nếu tất cả request đều về Tokyo:

		User Paris
		   ↓
		Tokyo Server
		   ↓
		User Paris


	Mỗi request phải đi hàng ngàn km.

	Kết quả:
		Website chậm
		Video giật
		Tăng tải server


	Tokyo Origin
		  ↓
	Sao chép dữ liệu
		Paris Edge
		London Edge
		New York Edge
		Singapore Edge


	Người dùng sẽ lấy dữ liệu từ Edge gần nhất.


# Bản chất kỹ thuật

	CDN hoạt động dựa trên Cache.
	
	##Cache Miss
		Lần đầu tiên:


User
 ↓
CDN
 ↓
Origin Server
 ↓
CDN Cache
 ↓
User

	CDN chưa có dữ liệu.
	Phải hỏi Origin.

	## Cache Hit

	Lần tiếp theo:
		User
		 ↓
		CDN Cache
		 ↓
		User

	Không cần Origin.
	
		
	Đây là mục tiêu lớn nhất của CDN:
	Tăng Cache Hit Ratio

	Cache Hit càng cao:
		Website càng nhanh
		Origin càng nhẹ
		Chi phí càng thấp	
	
	
	
	Ví dụ đời thường

	Giả sử Netflix.
		Không dùng CDN:

		100 triệu người
			  ↓
		Tokyo Datacenter

	Tokyo sẽ chết ngay lập tức.	
	
	
	
	Thực tế:

	Tokyo
	 ↓
	CDN toàn cầu
		Mỹ
		Đức
		Nhật
		Úc
		Việt Nam

	Mỗi khu vực xem từ cache gần nhất.

# CDN cache cái gì?	
	
	## Static Content
	Rất phù hợp

	Ví dụ:
		HTML
		CSS
		JavaScript
		Image
		Video
		PDF	
		
	
	
	##Dynamic Content
	
	Ví dụ:
		API
		Login Response
		User Profile

	AWS rất thích hỏi:
	Static content
	=> Nghĩ ngay CloudFront/CDN	
	
	
	
	
# 	Lợi ích thực sự của CDN
	
| Lợi ích           | Giải thích                     |
| ----------------- | ------------------------------ |
| Giảm Latency      | User lấy dữ liệu gần nhất      |
| Giảm tải Origin   | Ít request vào server          |
| Giảm Cost         | Giảm traffic từ Origin         |
| Tăng Scalability  | Chịu tải lớn                   |
| Tăng Availability | Một Edge lỗi vẫn còn Edge khác |
	
	
	
# 	Trade-off
	
	
	CDN không phải thuốc tiên.

	Ưu điểm
		Nhanh
		Rẻ
		Scale tốt
		
	Nhược điểm
		Dữ liệu có thể cũ
	
	Ví dụ:
		Origin cập nhật ảnh mới
		Nhưng Edge vẫn giữ ảnh cũ.
		Gọi là:
		Stale Cache	
		

# Cache Invalidation tốn thời gian

	Muốn xóa cache:
		Invalidate
	CDN phải cập nhật toàn cầu.

# CloudFront vs Global Accelerator

| CloudFront     | Global Accelerator |
| -------------- | ------------------ |
| Cache          | Có                 |
| CDN            | Có                 |
| Static Content | Tốt                |
| HTTP/HTTPS     | Chủ yếu            |



| Global Accelerator |       |
| ------------------ | ----- |
| Cache              | Không |
| CDN                | Không |
| TCP/UDP            | Tốt   |
| Gaming             | Tốt   |
| Real-time App      | Tốt   |
