# AWS EFS (Elastic File System)
	Amazon EFS là một hệ thống file (Network File System - NFS) được chia sẻ qua mạng, cho phép nhiều EC2 cùng đọc và ghi vào cùng một thư mục tại cùng một thời điểm.


	Đừng nhớ theo kiểu:
		"EFS là dịch vụ lưu trữ file."
		Điều đó quá nông.

	Hãy nhớ bản chất:
		EFS = Một ổ cứng dùng chung (Shared Disk) nằm trên mạng.
		Không thuộc riêng server nào.
		Không gắn chết vào một EC2.
		Bất kỳ EC2 nào được phép đều mount vào và nhìn thấy cùng dữ liệu.



	Giống như:

            Amazon EFS
        -----------------
       |  /images        |
       |  /videos        |
       |  /logs          |
        -----------------
          /      |      \
         /       |       \
      EC2-1    EC2-2    EC2-3

Tất cả đều thấy cùng file.




# 2. Ví dụ đời thường


Có một công ty có:
	10 nhân viên
	Nếu mỗi người có USB riêng

	A -> USB A
	B -> USB B
	C -> USB C

	Mỗi người sửa một file.
	Người khác không thấy.
	Đó giống EBS.



Nếu công ty có:

	NAS
			   NAS
		  -------------
		  report.xlsx
		  image.png
		  video.mp4
		  -------------

		 /   /   /   /
		A   B   C   D
	Ai mở cũng là cùng file.
	Đó chính là EFS.



# 3. Ví dụ System Design


	Website Upload ảnh.

	Có:
		ALB
		↓
		EC2-1
		EC2-2
		EC2-3

		Khách upload
		avatar.png
		Nếu dùng EBS
		EC2-1
		/avatar.png
		Request tiếp theo đi vào EC2-2
		404 Not Found
		Vì EC2-2 không có file.


	Nếu dùng EFS
	EC2-1

		   \
			EFS
		   /
	EC2-2
	Upload ở EC2-1
	↓
	EC2-2 nhìn thấy ngay.
	Không cần sync.


	Đây là lý do EFS cực kỳ phổ biến với:
		WordPress
		CMS
		Upload ảnh
		Shared configuration
		Shared log
		ML dataset

#	5. Các đặc điểm quan trọng 

	Shared Storage
	Nhiều EC2 dùng chung.


		NFS
		Mount bằng:
		mount

		Linux dùng rất nhiều.
		Windows không dùng.



		Elastic
		Không cần resize.
		Không cần:
		Increase Volume
		Dung lượng tăng tự động.
		Ví dụ
		10 GB
		↓
		50 GB
		↓
		200 GB
		↓
		5 TB
		Không cần thao tác gì.


		Multi AZ
		EFS lưu nhiều AZ.
		Một AZ chết
		↓
		EFS vẫn hoạt động.
		Đây là điểm khác cực lớn so với EBS.


		Managed
		Không quản RAID.
		Không quản replication.
		Không quản disk.
		AWS làm hết.



# 6. So sánh EFS vs EBS vs S3

| Tiêu chí                | EBS                           | EFS         | S3                    |
| ----------------------- | ----------------------------- | ----------- | --------------------- |
| Kiểu lưu trữ            | Block                         | File        | Object                |
| Mount như ổ cứng        | ✔                             | ✔           | ✖                     |
| Shared nhiều EC2        | ✖ (trừ Multi-Attach giới hạn) | ✔           | Không mount trực tiếp |
| Giao thức               | Block Device                  | NFS         | HTTP/API              |
| Auto Scaling dung lượng | ✖                             | ✔           | ✔                     |
| Multi AZ                | ✖                             | ✔           | ✔                     |
| Phù hợp                 | Database, OS                  | Shared File | Backup, ảnh, video    |



# 7. Trade-off
	Ưu điểm
		✅ Chia sẻ nhiều EC2
		✅ Auto Scale
		✅ Không cần resize
		✅ Highly Available
		✅ Multi AZ

	Nhược điểm

		Đắt hơn EBS.
		Latency cao hơn EBS.
		Không phù hợp Database IOPS cực cao.
		Không hỗ trợ Windows.


# 8. Khi nào dùng?

Dùng EFS khi:
	Nhiều EC2 cùng đọc ghi
	Shared upload
	WordPress
	CMS
	Home Directory
	ML Dataset
	Container shared storage

Không dùng EFS nếu:
	Chỉ có 1 EC2
	Database
	Cần latency thấp nhất


# 9. Các từ khóa xuất hiện trong đề thi

	Shared storage

	Concurrent access





# 10. Các bẫy (Trap)

| Đề bài                       | Đáp án đúng               | Bẫy                                |
| ---------------------------- | ------------------------- | ---------------------------------- |
| Một EC2 cần ổ đĩa nhanh nhất | EBS                       | Nhiều người chọn EFS vì "file"     |
| Nhiều EC2 cùng dùng dữ liệu  | EFS                       | Chọn EBS là sai                    |
| Lưu ảnh website              | S3 hoặc EFS (tùy yêu cầu) | Nếu cần mount như filesystem → EFS |
| Backup lâu dài               | S3                        | Không dùng EFS                     |
| Database                     | EBS                       | Không dùng EFS                     |
