# Định nghĩa bản chất

	AWS Lambda không phải là một server.
	AWS Lambda cũng không phải là một chương trình chạy liên tục.
	Lambda thực chất là:
		Một môi trường thực thi (Execution Environment) được AWS tạo ra tạm thời để chạy một đoạn code khi có Event xảy ra.

	Nói đơn giản:
		Event
			  ↓
		AWS tạo Runtime
			  ↓
		Load code
			  ↓
		Chạy function
			  ↓
		Trả kết quả
			  ↓
		Nếu không dùng nữa thì có thể hủy runtime

	Điểm quan trọng nhất:
		Lambda là Event-driven Compute.
			Không có Event
			↓
			Không có Lambda chạy
			↓
			Không tốn tiền compute.
			
		Đây chính là triết lý của Serverless.

# Điều AWS thực sự làm phía sau


	Khi Event tới:

	Ví dụ:

		S3 Upload
		AWS sẽ làm:
		1. Chọn Region
		↓
		2. Tìm Lambda Function
		↓
		3. Chuẩn bị Container
		↓
		4. Khởi động Runtime
		(Java/Python/Node...)
		↓
		5. Load code
		↓
		6. Chạy Handler
		↓
		7. Trả Response
		↓
		8. Giữ Container một thời gian
		↓
		9. Nếu có request mới
		Reuse Container
		Nếu lâu không dùng
		Destroy Container
		Đây gọi là:
		Execution Environment Lifecycle


# Warm Start và Cold Start

	Đây là phần AWS rất thích hỏi.
	Cold Start
	Container chưa tồn tại.
	AWS phải:
		Create Container
		↓
		Load Runtime
		↓
		Load Library
		↓
		Load Code
		↓
		Run

	Tốn thêm vài chục ms
	đến vài giây.
	Java thường chậm hơn Node.


	Warm Start
	Container vẫn còn.
	AWS chỉ:
		Reuse Container
		↓
		Run Handler
		Nhanh hơn rất nhiều.


	Ví dụ

	Lần đầu:
		API Gateway
		↓
		Lambda
		↓
		1.8s
		Sau 30 giây
		gọi lại
		API Gateway
		↓
		Lambda
		↓
		70ms
		Đó là Warm Start.


# AWS giữ Container bao lâu?

	Không có tài liệu công bố.
	AWS tự quyết định.

	Có thể:
		5 phút
		10 phút
		20 phút

	hoặc lâu hơn.
	Không nên code dựa vào giả định này.



# Điều gì được giữ lại sau Warm Start?

Ví dụ Java

public class Handler {
    static Database db = connect();
}

	Lần đầu
	connect DB
	Sau đó
	Warm Start
	Không connect lại.
	Reuse luôn.
	Đây là kỹ thuật tối ưu cực kỳ phổ biến.


# Nhưng điều gì KHÔNG được giữ?
	RAM
	File trong /tmp
	Connection

	Không có gì được đảm bảo.
	AWS có thể destroy container bất cứ lúc nào.

# Nhưng có giới hạn 


Account có Concurrency Limit.
	Ví dụ
	1000
	Request thứ
	1001
	↓
	Throttle.

	Reserved Concurrency
	Provisioned Concurrency
	để kiểm soát.
