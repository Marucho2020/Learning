# 1. Bản chất thật sự của Knowledge Bases là gì? 

	Định nghĩa sách giáo khoa
		Knowledge Bases for Amazon Bedrock giúp kết nối LLM với dữ liệu riêng của doanh nghiệp để thực hiện Retrieval-Augmented Generation (RAG).

	Đọc vậy gần như không hiểu.


# Định nghĩa bản chất

	Knowledge Base = "Bộ não tìm tài liệu trước khi AI trả lời" 


	Thay vì:
		User
		 ↓
		LLM
		 ↓
		Answer

	thì sẽ là:
		User
		 ↓
		Knowledge Base tìm dữ liệu liên quan
		 ↓
		LLM
		 ↓
		Answer

	AI không còn trả lời bằng kiến thức đã học sẵn nữa.

	Nó sẽ:
		Tìm tài liệu liên quan
		Lấy nội dung phù hợp
		Nhét vào Prompt
		Mới bắt đầu trả lời


# Ví dụ đời thường


	Bạn hỏi:
		Chính sách nghỉ phép của công ty là gì?

	LLM bình thường:
		Tôi không biết công ty bạn.

	hoặc
		Theo thông lệ thì...
		=> Hallucination.

	Knowledge Base:
		Employee Handbook.pdf
		đã được đưa vào Knowledge Base.

	AI sẽ:
		Search
		↓
		Tìm đúng đoạn nghỉ phép
		↓
		Đưa vào Prompt
		↓
		Trả lời

	Kết quả:
		Theo quy định công ty ABC,
		nhân viên được nghỉ 20 ngày/năm...


# 2. Tại sao AWS tạo ra Knowledge Bases?
	Vì vấn đề lớn nhất của AI doanh nghiệp là:


	AI không biết dữ liệu nội bộ

	Ví dụ:
		Confluence
		SharePoint
		S3
		PDF
		Word
		Wiki
		ServiceNow

	LLM không biết.

	Nếu Fine-tuning:
		Đắt
		Chậm
		Khó update

	Mỗi lần tài liệu đổi phải train lại.


	AWS chọn:
		RAG
		(Retrieval Augmented Generation)

	Thay vì:
		Train lại model

	thì:
		Search dữ liệu mới nhất


# 3. AWS Mapping
	Knowledge Base không phải một service độc lập.

| Thành phần   | AWS Service                    |
| ------------ | ------------------------------ |
| LLM          | Bedrock                        |
| Embedding    | Titan Embeddings               |
| Vector Store | OpenSearch / Aurora / Pinecone |
| Data Source  | S3                             |
| Query Engine | Knowledge Base                 |
| Security     | IAM                            |



			PDF
			DOCX
			HTML
			CSV
			↓
			S3
			↓
			Knowledge Base
			↓
			Embedding
			↓
			Vector Database
			↓
			Bedrock Model
			↓
			User

# 4. Ingestion là gì?
	Đề thi rất thích hỏi.


	Khi upload file:
		PDF 500 trang
		LLM không đọc trực tiếp.

	AWS sẽ:
		Chunking
		↓
		Embedding
		↓
		Store Vector


		Ví dụ

		PDF:

		500 trang

		AWS chia:
			Chunk 1
			Chunk 2
			Chunk 3
		...

		Mỗi chunk:
			Vector

		ví dụ:
			[0.22,0.18,0.97,...]

		Đây gọi là:
			Ingestion



# 5. Retrieval là gì? (truy xuất)


	Khi user hỏi:
		How many vacation days do employees get?

	Knowledge Base sẽ:

		Convert question
		↓
		Embedding
		↓
		Similarity Search
		↓
		Top K Results
		↓
		LLM



	Thực tế:

			Question
			↓
			Vector
			↓
			Nearest Neighbor Search
			↓
			Relevant Chunks
			↓
			Answer


# 6. RAG là gì?

		AWS thi cực thích.

		RAG:
			Retrieval
			+(And)
			Generation



		Không phải:
			User
			↓
			Model

		Mà:
			User
			↓
			Retrieve Documents
			↓
			Inject Context
			↓
			Generate Answer



		Lợi ích
			Giảm Hallucination
			AI bớt bịa.
			Dữ liệu mới nhất
			Không cần retrain.
			Rẻ hơn Fine-Tuning

		Fine-tuning:
			Train model

		RAG:
			Search data


# 7. So sánh Fine-Tuning vs Knowledge Base

| Tiêu chí                  | Fine-Tuning        | Knowledge Base   |
| ------------------------- | ------------------ | ---------------- |
| Học dữ liệu mới           | Có                 | Không            |
| Trả lời theo tài liệu mới | Kém                | Tốt              |
| Chi phí                   | Cao                | Thấp             |
| Update dữ liệu            | Khó                | Dễ               |
| Hallucination             | Cao hơn            | Thấp hơn         |
| AWS khuyến nghị           | Không phải lúc nào | Rất thường xuyên |



# Góc nhìn Senior Architect

	Knowledge Base không làm AI thông minh hơn.
	Nó làm AI: Ít bịa hơn

	Đây là điểm rất nhiều người hiểu sai.
	Fine-tuning giúp model thay đổi hành vi.
	Knowledge Base giúp model có thêm dữ liệu để tham khảo.
	Trong doanh nghiệp Nhật (ngân hàng, bảo hiểm, ServiceNow, ERP, CRM), 80-90% use case GenAI thực tế hiện nay đang đi theo hướng:

	Bedrock
	+
	Knowledge Bases
	+
	S3
	+
	OpenSearch Vector
	chứ không phải train lại model.
