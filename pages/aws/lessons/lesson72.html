# Định nghĩa ngắn gọn 
	OpenSearch Vector Search là chức năng cho phép tìm kiếm dựa trên ý nghĩa (semantic meaning) thay vì chỉ tìm kiếm dựa trên từ khóa.



	Thay vì:
		"Tìm tài liệu chứa từ AWS"

	thì sẽ là:
		"Tìm tài liệu có nội dung liên quan tới AWS"

	ngay cả khi trong tài liệu không hề xuất hiện từ "AWS".


	## Bản chất thật sự

			Thông thường Search Engine hoạt động như:
				Keyword Search

			Ví dụ:

				Query:
				"How to deploy Spring Boot"

				Document A:
				"How to deploy Spring Boot on EC2"

				=> Match

			Nhưng:

				Query:
				"Deploy Java Application"

				Document A:
				"How to deploy Spring Boot on EC2"

					=> Có thể không match tốt
					vì từ ngữ khác nhau.



		Vector Search giải quyết bằng cách:

		Bước 1
			Embedding Model
			(Titan Embeddings, Cohere Embeddings...)
			chuyển text thành vector.

		Ví dụ:
			"Deploy Spring Boot"
			↓
			[0.21, -0.55, 0.91, ...]

			"Deploy Java Application"
			↓
			[0.23, -0.52, 0.89, ...]



				Hai vector này sẽ nằm gần nhau trong không gian toán học.
				Cosine Similarity
					0.97
				=> Ý nghĩa giống nhau.


		Bước 2

		Lưu vector vào OpenSearch
			Ví dụ:

| Document | Vector         |
| -------- | -------------- |
| Doc A    | [0.21,0.55...] |
| Doc B    | [0.44,0.22...] |
| Doc C    | [0.91,0.11...] |


		Bước 3
			User Query
			"How can I deploy Java service?"


			↓
			Embedding
			↓
			Vector
			↓
			OpenSearch tìm các vector gần nhất
			↓
			Top K Results



		Tóm tắt bản chất
			OpenSearch Vector Search thực chất là:
			Nearest Neighbor Search


			hay:
				Tìm vector gần nhất

			KHÔNG phải:
				Tìm keyword giống nhất



# 2. Mapping trong AWS
	Kiến trúc AWS rất hay ra đề:

User Query
      ↓
Titan Embeddings
      ↓
Vector
      ↓
OpenSearch Vector Index
      ↓
Top K Documents
      ↓
Claude / LLM
      ↓
Answer



	Dịch vụ AWS tương ứng

| Thành phần      | AWS Service              |
| --------------- | ------------------------ |
| Embedding Model | Bedrock Titan Embeddings |
| Vector Database | OpenSearch               |
| LLM             | Claude / Titan Text      |
| Orchestration   | Bedrock Knowledge Base   |
| Data Source     | S3                       |



# 3. OpenSearch Vector Search trong RAG
	Đây là thứ xuất hiện cực kỳ nhiều trong đề thi AI Specialty và các hệ thống AI hiện đại.

Ví dụ
	S3 chứa:

		Employee Handbook
		HR Policy
		Security Guideline

	User hỏi:
		Can I work remotely from overseas?


	Knowledge Base:

	Step 1

		Convert Query
		↓
		Embedding
		↓
		Vector



	Step 2
		Search OpenSearch
		↓
		Tìm document gần nhất


	Step 3

		Trả về
			Remote Work Policy


	Step 4
		Claude đọc tài liệu
		↓
		Sinh câu trả lời



	Đây chính là:
		RAG
		=
		Embedding
		+
		Vector Search
		+
		LLM




# 4. Vì sao AWS thích OpenSearch?


	Trước đây:
		Database
		không tối ưu cho vector.

	Ví dụ:
		RDS
		Aurora
		DynamoDB

		đều không sinh ra để tìm kiếm vector.


	OpenSearch hỗ trợ:
		Approximate Nearest Neighbor (ANN)

	cho phép:
		Tìm vector gần nhất
		trong hàng triệu vector.
		rất nhanh.




	Nếu có:

		10 triệu documents
	thì brute force:
		so sánh từng vector

		sẽ cực chậm.


	OpenSearch dùng:
		HNSW
		(Hierarchical Navigable Small World)
		để tăng tốc ANN Search.



# 5. So sánh dễ nhầm

|                                 | Keyword Search | Vector Search |
| ------------------------------- | -------------- | ------------- |
| Dựa trên                        | Từ khóa        | Ý nghĩa       |
| Elastic/OpenSearch truyền thống | ✓              | ✗             |
| Semantic Search                 | ✗              | ✓             |
| Synonym                         | Kém            | Tốt           |
| AI Search                       | Kém            | Tốt           |





	Ví dụ

	Query:
		Car

	Document:
		Automobile

	Keyword Search
	Không tìm thấy
	Vector Search
	Tìm thấy
	vì ý nghĩa giống nhau.



# 6. Trade-off

	Ưu điểm
		Semantic Search
		Hiểu ngữ nghĩa.
		Tốt cho RAG
		Hiện nay gần như tiêu chuẩn.
		Scale lớn
		Hàng triệu document.


	Nhược điểm
		Tốn chi phí Embedding
		Mọi document phải được embed trước.
		Không chính xác tuyệt đối
		Vector Similarity là:
			Gần đúng
			không phải:
			100% đúng

		Storage tăng
			Ngoài document còn phải lưu vector.
