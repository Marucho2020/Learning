# 1. High Availability (HA)

	High Availability = Hệ thống luôn sẵn sàng “gần như mọi lúc”	
		Nếu có lỗi → hệ thống vẫn chạy, nhưng có thể bị gián đoạn rất ngắn.


# Bản chất

		- Giảm downtime (thời gian chết)
		- Không cần 100% uptime
		- Cho phép fail + recovery nhanh


# Ví dụ thực tế

	Bạn có 2 server:
		- Server A (chính)
		- Server B (dự phòng)

	Khi A chết:
		→ chuyển sang B (mất vài giây)
	
	User có thể bị:
		reload lại trang
		request bị fail 1 lần	
	
	=> Đây là High Availability	
	

# Trong AWS

	Thường thiết kế kiểu:
		deploy nhiều instance trên nhiều AZ

	Ví dụ:
		Amazon EC2 ở 2 AZ
		Elastic Load Balancing để phân phối traffic



	Ví dụ kiến trúc

		  User
			↓
		Load Balancer
		 ↓        ↓
		AZ1       AZ2
		EC2       EC2


	Nếu AZ1 chết:
		→ traffic chuyển sang AZ2



# Ví dụ đời thường

	Giống như:
		bạn có 2 cửa hàng
		1 cái đóng → khách qua cái còn lại
		
	=> vẫn bán được, nhưng hơi gián đoạn




------------------ 日本語----------

**High Availability（高可用性）**とは：
		システムがほぼ常に利用可能である状態



# ポイント 	

	ダウンタイムを最小化する
	完全に停止しないわけではない
	障害後にすぐ復旧できる



# 例
	サーバーAが落ちたら：
		→ サーバーBに切り替え

	数秒の影響あり


# AWSでの実装
	Amazon EC2 を複数AZに配置
	Elastic Load Balancing を使用
