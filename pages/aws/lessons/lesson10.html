# Fault Tolerance

	Fault Tolerance = Hệ thống vẫn hoạt động ngay cả khi có lỗi xảy ra	
		Không gián đoạn. Không downtime. Không ai biết là có lỗi xảy ra.



# Bản chất

	- Không được phép gián đoạn
	- Không mất dữ liệu
	- hệ thống vẫn chạy bình thường


# Ví dụ thực tế
	2 server chạy song song:
		Server A + Server B (active-active)

	Nếu A chết:
		→ B vẫn xử lý ngay lập tức
		→ user KHÔNG bị ảnh hưởng

	=> không cần failover → vì đã chạy song song

# Khác biệt cực quan trọng

	High Availability: fail → rồi recover
	Fault Tolerance: fail → nhưng không ai nhận ra



# Trong AWS


Thiết kế kiểu:

	multi-AZ
	replication
	active-active
	
		Amazon RDS Multi-AZ
		Amazon S3 (replicate nhiều AZ)

# Ví dụ đời thường 

	Máy bay:
		có 2–3 động cơ
		1 cái hỏng → vẫn bay bình thường
	=> đó là Fault Tolerance



------------ 日本語 -----------

Fault Tolerance（耐障害性）**とは：障害が発生してもシステムが止まらずに動き続ける能力


# ポイント

	- ダウンタイムなし
	- ユーザーに影響なし
	- 常に稼働し続ける

# 例
		サーバーA + サーバーB（同時稼働）
	
	Aが落ちても：
		→ Bがそのまま処理
		ユーザーは気づかない
	

AWSサービス例

	Amazon RDS（Multi-AZ）
	Amazon S3（自動レプリケーション）
