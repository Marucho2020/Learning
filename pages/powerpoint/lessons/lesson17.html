Lesson 16 -- この機能で何ができるか -//

	「この機能で何ができるか」
	
	
# 1. Ý nghĩa trong bối cảnh IT	
	
	「この機能で何ができるか」
		→ Trả lời: User dùng cái này thì làm được việc gì cụ thể trong nghiệp vụ?	
		
		Không phải:

		❌ Nó có field gì
		❌ Nó chạy logic gì	
			
	Mà là:
		✅ Nó giúp ai làm gì nhanh hơn / đúng hơn / ít lỗi hơn	
		
		
	
	
# 2. Ví dụ thực tế (ServiceNow) 	
	Chức năng: Incident Auto Assignment	
	
	👉 Không phải nói:
		system dùng rule để assign ticket

	👉 Mà phải nói:
		Giúp ticket được tự động phân công đúng bộ phận ngay khi tạo	
		
	
# 3. Đoạn mô tả kiểu tài liệu (報告書) 	
	
	本機能により、インシデント登録時に内容およびカテゴリに応じて担当グループが自動的に割り当てられる。
	これにより、手動による振り分け作業が不要となり、対応開始までのリードタイムを短縮するとともに、誤
	った担当割り当てによる対応遅延を防止できる。	
	
	👉 Nhìn kỹ:

		Không nói technical
		Chỉ nói user impact + business value	
		
		
#4. Slide PowerPoint (bullet)
	
	- インシデント登録時に担当グループを自動割当
	- 手動振り分け作業を削減
	- 初動対応までの時間を短縮
	- 誤割当による対応遅延を防止	
	
	
# 	5. Tiêu đề slide chuẩn Nhật

	👉 Đừng viết:
		❌ 自動アサイン機能について

	👉 Viết kiểu senior:
		✅ インシデント自動割当により対応迅速化を実現



# 6. NG vs OK 

	❌ NG (junior)
		この機能では、条件に応じて担当者を自動的に設定することができます。

		👉 Vấn đề:
			- Mơ hồ (“できます”)
			- Không nói lợi ích
			- Không nói impact


	✅ OK (chuẩn Nhật)
	
	本機能により、インシデント登録時に担当グループが自動的に割り当てられ、手動対応を削減し、
		対応開始までの時間短縮を実現する。
		
			👉 Điểm mạnh:
				Có context (khi nào)
				Có action (làm gì)
				Có value (giảm gì, nhanh gì)


		Chốt (quan trọng)
			Nếu bạn viết mà đọc xong vẫn phải hỏi:
			→ “Rồi user dùng để làm gì?”
			=> Bạn viết sai.

			Câu này bản chất là:
				Translate system → business value
				
			Không làm được cái này, bạn sẽ mãi đứng ở level implementer, không bao giờ lên được architect.



# Bài tập 

	Bối cảnh (ServiceNow / SaaS)

	Bạn đang thiết kế chức năng:

	👉 承認リマインド機能（Approval Reminder）
		- Khi user gửi request nhưng manager chưa approve
		- Sau X giờ / X ngày → system tự động gửi reminder
		- Có thể gửi nhiều lần cho đến khi approve

	✍️ Yêu cầu bạn viết (rất quan trọng)
		Viết 1 đoạn ngắn (3–5 dòng) theo format:
		👉 「この機能で何ができるか」

	⚠️ Rule (tôi sẽ chấm rất gắt)

	Phải có đủ 3 yếu tố:
		- いつ (khi nào / context)
		- 何が起きる (system làm gì)
		- どんな価値がある (business value)
		
	❌ Những thứ tôi sẽ trừ điểm nặng
		Chỉ mô tả technical (メール送信できます)
		Không có value (giảm gì? nhanh gì?)
		Dùng câu mơ hồ kiểu 「〜が可能です」
		Không rõ ai hưởng lợi	


		
	## Mô tả 
		本機能により、申請後一定時間承認が行われていない場合、
		システムが承認者へ自動的にリマインド通知を送信する。
		さらに、リマインド後も未承認の状態が継続する場合、通知頻度および通知先を段階的に拡張し、
		メールやチームチャットを通じて再通知を行う。

		これにより、承認対応の見落としを防止し、承認遅延およびSLA違反のリスクを低減するとともに、
		業務全体のリードタイム短縮を実現する。


	## Bullet
		- 申請後、一定時間未承認の場合に自動リマインド通知を送信
		- 未承認が継続する場合、通知頻度・通知先を段階的に拡張
		- メールおよびチームチャットによる再通知を実施
		- 承認対応の見落としを防止
		- 承認遅延およびSLA違反リスクを低減
		- 業務全体のリードタイムを短縮


		
		
		
	Tóm tắt : 
	
	承認見落としを防止するため、未承認状態に応じた段階的な通知エスカレーションを実施し、
	承認遅延の抑制およびリードタイム短縮を実現する。

		- mình dùng pattern gì để giải?
		- nó tạo ra impact gì?
