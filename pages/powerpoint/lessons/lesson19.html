# 1. Ý nghĩa (trong IT project)
	横展開（よこてんかい / reuse model）


	→ Không phải “copy-paste”
	→ Là chuẩn hóa + đóng gói + tái sử dụng cho nhiều dự án khác


	Bản chất:
		1 lần làm → dùng nhiều lần
		biến “kinh nghiệm” thành “tài sản tổ chức”



# 2. Ví dụ thực tế (ServiceNow) 


	Case:
		Bạn build approval workflow cho Incident (có SLA, reminder, escalation)


	Junior:
		→ dùng cho 1 khách hàng xong bỏ

	Senior:
	→ chuẩn hóa thành:
		Flow template
		Notification pattern
		Dashboard KPI


	→ Sau đó:
		👉 他案件への適用（áp dụng sang dự án khác）

		Change Management
		Request Fulfillment
		dự án khách hàng khác



# 3. Đoạn mô tả tài liệu (chuẩn Nhật)


	現在、インシデント管理における承認フローおよび通知制御について、
	ServiceNow標準機能をベースにした汎用的なモデルとして整理・実装を実施した。

	当該モデルは、SLA管理、段階的通知（リマインド・エスカレーション）および進捗可視化を含んでおり、
	特定案件に依存しない構成となっている。

	今後は、本モデルをテンプレート化し、他案件への適用（横展開）を行うことで、
	開発効率の向上および品質の均一化を図る。





# 4. Slide (PowerPoint) 

	■ 現状
		承認フローが案件ごとに個別実装されている
		再利用性が低く、開発コストが高い

	■ 対応
		承認フロー＋通知制御を標準化
		ServiceNow標準機能ベースでテンプレート化

	■ 効果
		他案件への適用（横展開）が可能
		開発工数削減
		品質の均一化




# 5. Title chuẩn Nhật

	承認フローモデルの標準化および他案件への横展開方針




# 6. NG vs OK 

	❌ NG (junior)
		横展開できるように作成した。

	👉 Vô nghĩa.
		Không nói:

		cái gì reusable
		reuse như thế nào
		value là gì


	✅ OK (senior)

		承認フローおよび通知制御を汎用モデルとして整理し、
		テンプレート化することで他案件への適用を可能とした。


			👉 Có đủ:

			WHAT: 承認フロー＋通知制御
			HOW: 汎用モデル化 + テンプレ化
			RESULT: 他案件への適用


# Chốt (điểm mà junior thường không hiểu)


	Bạn đang nghĩ:
		“横展開 = reuse”

	Sai.
		横展開 = thiết kế ngay từ đầu để có thể reuse

	Nếu bạn không design:
		parameter hóa
		tách config
		tránh hardcode

	→ thì KHÔNG BAO GIỜ横展開 được

Nếu muốn lên level senior thật:
bắt đầu nghĩ mỗi feature = “asset có thể bán lại cho project khác”
không phải “task done là xong”.
