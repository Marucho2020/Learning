# Khái niệm 
	AngularJS Validation – Kiểm tra dữ liệu nhập (form validation)
	Validation trong AngularJS cho phép bạn: 
		- Kiểm tra giá trị đầu vào của người dùng. 
		- Hiển thị cảnh báo hoặc thông báo lỗi nếu dữ liệu không hợp lệ 
		- Kiểm soát việc bật/tắt nút submit dựa trên tính hợp lệ của form. 
		AngularJS cung cấp sẵn nhiều directive kiểm tra đầu vào như: required, minlength, maxlength, type=email, v.v.
		
	
# 🧪 Ví dụ cơ bản



  
  
    Tên người dùng là bắt buộc
  



	##✅ Giải thích:
		name="myForm": Đặt tên form để truy cập từ AngularJS.
		ng-model="user": Gắn dữ liệu vào biến user.
		ng-show="myForm.user.$invalid": Chỉ hiện cảnh báo nếu field không hợp lệ.


# 📍 Các thuộc tính trạng thái trong AngularJS 

| Thuộc tính   | Ý nghĩa                        |
| ------------ | ------------------------------ |
| `$valid`     | Đúng định dạng                 |
| `$invalid`   | Sai định dạng                  |
| `$touched`   | Đã từng focus vào rồi rời khỏi |
| `$untouched` | Chưa từng đụng vào             |
| `$dirty`     | Giá trị đã bị thay đổi         |
| `$pristine`  | Giá trị chưa thay đổi          |


# 🎛 Dùng để disable nút Submit khi dữ liệu chưa hợp lệ 



# 📦 Các directive kiểm tra thông dụng

	| Directive          | Ý nghĩa              |
| ------------------ | -------------------- |
| `required`         | Bắt buộc phải nhập   |
| `ng-minlength="x"` | Độ dài tối thiểu     |
| `ng-maxlength="x"` | Độ dài tối đa        |
| `type="email"`     | Định dạng email      |
| `type="number"`    | Chỉ cho phép nhập số |



# 💡 Ví dụ đầy đủ



  
  
    Email không hợp lệ.
  

  




# ⚠️ Lưu ý thực tiễn

| Việc làm                                      | Có nên không? | Giải thích                                                  |
| --------------------------------------------- | ------------- | ----------------------------------------------------------- |
| Kiểm tra phía client bằng AngularJS           | ✅             | Nhanh, tiện                                                 |
| Bỏ qua kiểm tra phía server                   | ❌             | **Bắt buộc phải kiểm tra cả phía backend**, tránh bị bypass |
| Dùng `$touched` để tránh hiện lỗi ngay từ đầu | ✅             | Tránh gây khó chịu cho người dùng                           |


# 🔧 Ngoài ra:

Bạn có thể viết custom validation directive nếu muốn kiểm tra logic đặc biệt như: "phải khác mật khẩu cũ", hoặc "số điện thoại Việt Nam hợp lệ", v.v.

# ✅ Kết luận 

	-  AngularJS validation hoạt động tốt và đầy đủ cho phần lớn nhu cầu.
	-  Kết hợp thông minh giữa ng-show, $touched, $invalid, ng-disabled sẽ giúp trải nghiệm UX tốt hơn.
	-  Validation client chỉ là tuyến đầu – backend luôn phải kiểm tra lại.
