Lesson 6 -- Data Binding -//

# Khái niệm 
	Nó là cơ chế tự động đồng bộ dữ liệu giữa:
		Model (dữ liệu trong controller)
		và View (giao diện HTML)



# Có mấy loại Data Binding? 
	## 🔹 1. One-Way Binding (1 chiều)
		Từ Controller → View	
		Sử dụng dấu {{ }} hoặc các directive như ng-bind.
		
			
{{ name }}
     
			
  

		✔ Khi controller cập nhật name, UI tự thay đổi.
		✘ Nhưng nếu user thay đổi trên UI, controller không nhận được.
	
	
	
	## 🔹 2. Two-Way Binding (2 chiều)
		Sử dụng ng-model
		
		
		
{{ name }}


		✔ Khi user gõ vào input → biến name trong controller thay đổi
		✔ Khi controller thay đổi name → input cũng cập nhật
		
		→ Đây là kiểu rất mạnh mẽ và là điểm khác biệt so với các thư viện khác lúc bấy giờ.	
	
	
	
# So sánh One-way vs Two-way 
	| Loại         | Cập nhật từ View → Model | Cập nhật từ Model → View |
| ------------ | ------------------------ | ------------------------ |
| `{{ name }}` | ❌                        | ✅                        |
| `ng-bind`    | ❌                        | ✅                        |
| `ng-model`   | ✅                        | ✅                        |
	
	
	
# 	🧨 Cạm bẫy thực tế 
Nếu bạn binding biến xử lý logic (như $ctrl.isSearched) bằng ng-model, thì mỗi khi user thay đổi input là controller cũng thay đổi — gây bug, điều kiện ng-if, ng-disabled chạy sai.

→ Lời khuyên: luôn dùng biến tạm cho input, chỉ copy sang biến xử lý khi user ấn nút hoặc thực hiện hành động.	
	
	
# 	✅ Kết luận 

	{{ }}: chỉ để hiển thị
	ng-bind: giống {{ }}, nhưng tránh bị flash khi trang load (Flicker)
	ng-model: để đọc và ghi dữ liệu từ UI
