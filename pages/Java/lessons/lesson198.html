Lesson 198 -- Function Composition & Currying -//

# 🧩 1. Function Composition — “Ghép hàm như pipeline”

	Function composition = kết hợp nhiều hàm nhỏ (pure functions) thành một hàm lớn, nơi output của hàm trước là input của hàm sau.

	Trong toán học: (f ∘ g)(x) = f(g(x))
	Trong Java: f.compose(g) hoặc f.andThen(g)


	## 🔹 Hai hướng ghép trong Java: 

		Function getName = Employee::getName;
		Function toEmailFormat = name -> name.toLowerCase() + "@company.com";

	👉 andThen – chạy f rồi g:
	
		Function emailFn = getName.andThen(toEmailFormat);

		andThen: f(x) → g(f(x))
		Thứ tự chạy: getName → toEmailFormat.

	👉 compose – chạy g rồi f:

		Function parseInt = Integer::parseInt;
		Function isEven = n -> n % 2 == 0;
		
		Function checkEven = isEven.compose(parseInt);


	## 🔹 Enterprise hình dung:
	
	Thay vì code procedural:
		String email = toEmailFormat(getName(emp));


	ta dùng composition:
		Function emailFn = getName.andThen(toEmailFormat);

	→ Ta đóng gói cả logic pipeline vào một function tái sử dụng được.
	Có thể inject, chain, hoặc test riêng từng bước.



# 🧱 2. Chuyển sang tư duy “lego logic”


	Thay vì viết 1 function dài 200 dòng,
	senior functional design chia nhỏ:

| Step                   | Mục tiêu                | Kiểu function |
| ---------------------- | ----------------------- | ------------- |
| validate input         | Predicate     | boolean       |
| transform data         | Function | object        |
| compute business value | Function   | object        |
| output                 | Consumer        | void          |


	Rồi ghép lại:
		→ Mỗi khối pure, testable, reusable.

Function pipeline =
    validate
      .andThen(transform)
      .andThen(compute);




# 🧮 3. Currying — “Biến hàm n tham số thành chuỗi hàm 1 tham số”

	Currying = biến hàm nhiều tham số thành chuỗi các hàm nhận 1 tham số và trả về 1 hàm mới.

	## Ví dụ: 
		BiFunction add = (a, b) -> a + b;

	Currying thủ công:	
	
Function> curriedAdd =
    a -> (b -> a + b);


	Sử dụng:

		Function add5 = curriedAdd.apply(5);
		int result = add5.apply(3); // = 8
	→ Ta “ghim” trước một phần tham số để tạo function mới — gọi là partial application.


	## 💡 Enterprise ý nghĩa: 

Function> buildValidator =
    field -> (emp -> validateField(emp, field));




# 🏢 4. Enterprise Example: Validation & Transformation Pipeline

	Check employee valid
	Transform thành DTO
	Format email

	## Functional style:

Predicate isActive = Employee::isActive;
Function toDTO = emp -> new DTO(emp.getId(), emp.getName());
Function toEmail = dto -> dto.name().toLowerCase() + "@corp.com";

Function pipeline =
    toDTO.compose(isActiveFilter(isActive)).andThen(toEmail);


	## Hoặc tách dynamic parameter bằng currying:





	→ Functional style cho phép tạo logic động, thay đổi runtime mà không viết if/else spaghetti.

Function> hasField =
    field -> emp -> !getField(emp, field).isEmpty();

Predicate hasEmail = hasField.apply("email");




# ⚙️ 5. Khác biệt tư duy: OOP vs Functional Composition


| So sánh         | OOP Layered                   | Functional Pipeline   |
| --------------- | ----------------------------- | --------------------- |
| Cấu trúc        | Class, method                 | Function chain        |
| Tổ chức         | Service → Mapper → Repository | map → filter → reduce |
| State           | mutable object                | immutable data        |
| Kết hợp         | Call tuần tự                  | Compose logic         |
| Test            | cần mock dependency           | test từng function    |
| Tái sử dụng     | kế thừa (inheritance)         | composition           |
| Dễ parallel hóa | Khó (shared state)            | Dễ (pure)             |




# ⚔️ 6. Khi nào nên dùng functional composition

| Tình huống                                  | Nên dùng functional                   |
| ------------------------------------------- | ------------------------------------- |
| **Business logic phức tạp, nhiều rule**     | ✅ compose dễ đọc, test tốt            |
| **Data processing, analytics, ETL, stream** | ✅ pipeline cực tự nhiên               |
| **Reactive flow (WebFlux, Kafka stream)**   | ✅ native pattern                      |
| **Stateful API, transaction, workflow**     | ⚠️ vẫn cần OOP layer để quản lý state |

🧠 Triết lý: Functional để build logic, OOP để model world.




# 🧩 7. Liên hệ với các ngôn ngữ khác

| Ngôn ngữ       | Composition tools                 | Currying support                |
| -------------- | --------------------------------- | ------------------------------- |
| **JavaScript** | `pipe()`, `compose()` (Ramda)     | native function return function |
| **Kotlin**     | `let`, `run`, `also`              | hỗ trợ trực tiếp                |
| **Scala**      | `andThen`, `compose`              | built-in currying               |
| **Haskell**    | `.` (compose)                     | currying mặc định               |
| **Python**     | `functools.partial`               | simulate currying               |
| **Java**       | `Function.andThen()`, `compose()` | currying thủ công               |
