Lesson 10 -- Đa kế thừa -//

	# ❌ Đa kế thừa class  
		❌ Java không hỗ trợ đa kế thừa class vì "diamond problem" và xung đột trạng thái (state conflict).
		
class A {
    void say() { System.out.println("A"); }
}

class B extends A {
    void say() { System.out.println("B"); }
}

class C extends A {
    void say() { System.out.println("C"); }
}

// class D extends B, C {} // ❌ Không được – Java không biết chọn "say()" của B hay C


	# ✅ đa kế thừa interface 
		## 1. Interface không có state → tránh xung đột biến (trước Java 8) 
		Trước Java 8, interface chỉ có:
			- method abstract
			- hằng số public static final
			- Nên không có xung đột dữ liệu giữa các interface → an toàn để đa kế thừa.
		
		## 2. Từ Java 8, interface có default method → có thể có logic, nhưng vẫn không gây mâu thuẫn vì quy tắc rõ ràng:
		
		Nếu có 2 interface cùng có default method trùng tên, class phải override rõ ràng:

interface A {
    default void say() { System.out.println("A"); }
}

interface B {
    default void say() { System.out.println("B"); }
}

class C implements A, B {
    public void say() {
        A.super.say(); // hoặc B.super.say()
    }
}



# 🧠 Tổng kết:


| So sánh                   | Class                                | Interface                                         |
| ------------------------- | ------------------------------------ | ------------------------------------------------- |
| Có thể đa kế thừa?        | ❌ Không                              | ✅ Có                                              |
| Có state (biến instance)? | ✅ Có                                 | ❌ Không (chỉ hằng số)                             |
| Có logic sẵn?             | ✅ Có                                 | ✅ Từ Java 8 (default/static method)               |
| Có thể xung đột logic?    | ✅ Có (và Java không biết giải quyết) | ✅ Có – nhưng Java bắt buộc override để giải quyết |
