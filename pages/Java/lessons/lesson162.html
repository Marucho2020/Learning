Lesson 162 -- Thread Basics -//

# 🔹 Mục tiêu
	Hiểu cách tạo luồng, điều phối luồng, và quản lý luồng hiệu quả trong Java, tránh rơi vào bẫy tạo luồng bằng tay như junior-level.


# 🔧 1. Thread vs Runnable – sự khác biệt bản chất

	## Thread: 
		- Là class thực thi luồng.
		- Có thể override run(), nhưng đã bị xem là low-level, ít kiểm soát hơn Executor.
	
Thread t = new Thread(() -> System.out.println("Hello from Thread!"));
t.start();


	## Runnable:
		- Là Functional Interface, không trả về gì (void run()).
		- Dùng để tách logic khỏi quản lý luồng, giúp dùng với Executor dễ hơn.

	⛔ Tránh kế thừa trực tiếp Thread, vì giới hạn kế thừa đơn.




# 🔄 2. Callable & Future – xử lý kết quả bất đồng bộ

	## Callable: 
	
		- Functional interface như Runnable, nhưng trả về kết quả (T call() throws Exception).
		- Dùng cho tác vụ cần kết quả.

	## Future:
		- Đại diện cho kết quả tương lai của 1 tác vụ Callable.
		- Có thể get() để chờ kết quả, hoặc cancel() để dừng.

Callable task = () -> "Result";
ExecutorService executor = Executors.newSingleThreadExecutor();
Future future = executor.submit(task);
String result = future.get(); // Blocking



# ⚙️ 3. Executors – quản lý thread chuyên nghiệp

	##  Tại sao dùng Executor? 

		- Thread pooling: tái sử dụng luồng -> tiết kiệm CPU.
		- Quản lý lifecycle: shutdown, submit, monitor, scaling.
		- Tách biệt logic công việc khỏi logic quản lý tài nguyên

ExecutorService executor = Executors.newFixedThreadPool(5);
executor.submit(() -> doWork());
executor.shutdown();



# 🧠 4. ThreadFactory – kiểm soát chi tiết việc tạo luồng


	## Giúp bạn:

		- Đặt tên luồng dễ debug.
		- Set daemon flag, priority...
		- Tích hợp tốt với frameworks.

ThreadFactory namedFactory = r -> {
    Thread t = new Thread(r);
    t.setName("MyApp-Worker");
    return t;
};
ExecutorService executor = Executors.newFixedThreadPool(2, namedFactory);



# 💥 5. ThreadPoolExecutor – cấu hình chuyên sâu

	Khi bạn cần:

		- Kiểm soát hàng đợi.
		- Set core/max pool size.
		- Tự xử lý khi queue đầy (RejectedExecutionHandler).

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    2, 4,
    60, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100),
    new CustomThreadFactory(),
    new ThreadPoolExecutor.AbortPolicy()
);




# 🛠 Senior Tips

	- ✅ Đừng dùng new Thread(...) trừ khi bạn đang học/làm demo.
	- ✅ Luôn dùng Executors, nhưng khi vào production thì nên tự tạo ThreadPoolExecutor thay vì Executors.newFixedThreadPool vì nó ẩn quá nhiều.
