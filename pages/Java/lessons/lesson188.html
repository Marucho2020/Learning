Lesson 188 -- map vs flatMap trong Stream -//

	
# 1. map() vs flatMap() trong Stream	
	
	map(): biến đổi từng phần tử thành một "đối tượng khác", kết quả thường là Stream> nếu map ra một stream.	
		👉 ví dụ: dept.map(Department::getEmployees) → Stream>.
	
	flatMap(): vừa map, vừa "flatten" (フラット化), ép tất cả "tầng" xuống còn 1 cấp.
	
	👉 ví dụ: dept.flatMap(d -> d.getEmployees().stream()) → Stream.
	
	⚠️ Điểm senior cần nhớ: flatMap chính là cách "nối pipeline" tránh nesting rối rắm. Nó là chìa khóa để làm việc với data nhiều tầng (tree, graph, object nesting).
	
	
	
# 	2. Optional.map vs Optional.flatMap
	Optional trong Java chính là một monad container giống Stream nhưng chỉ chứa 0 hoặc 1 giá trị.
	
		Optional.map(): biến đổi giá trị bên trong, kết quả bọc thêm một Optional mới.
			👉 employee.map(Employee::getManager) → Optional> (lồng nhau).
	
		Optional.flatMap(): mở gói một lớp, tránh Optional>.
		👉 employee.flatMap(Employee::getManager) → Optional.
	
		⚠️ Senior point:
		Optional.map tạo nested container (入れ子コンテナ), Optional.flatMap phá bỏ nesting, giữ pipeline gọn.
	
	
	
# 3. Enterprise example	
	Giả sử có Department → Employees → Manager.	
		map: nếu dùng map liên tiếp, anh sẽ có 
		Stream> hoặc Optional>.
		
		flatMap: gộp tất cả nhân viên thành một stream duy nhất để tiếp tục pipeline.
	
		Trong Optional: employee.flatMap(Employee::getManager).map(Manager::getEmail) → xử lý an toàn null-safe pipeline cho email của Manager.
	


# 4. Pipeline tư duy chung
	Trong lập trình nói chung:

		map = apply transformation, có thể sinh ra layer mới.
		
		flatMap = map + flatten, giữ pipeline phẳng, dễ thao tác.
		
		Trong Java Stream/Optional: đây chính là cách chúng ta control data flow, không để rối bởi nested containers.


# 👉 Nói ngắn gọn kiểu senior:

	map = transform (変換).
	
	flatMap = transform + flatten (変換＋平坦化).
	
	Optional + Stream đều là monadic pipelines: map giữ container, flatMap tháo container.
