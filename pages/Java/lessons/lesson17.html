Lesson 17 -- Observable Module 観測可能なモジュール -//

				
# Khái niệm 
	Observable Module (観測可能なモジュール)				
	Khái niệm 観測可能なモジュール trong Java có thể dịch là "observable module" — tức là một module có thể quan sát được từ bên ngoài.			
				
	Trong hệ thống module của Java (giới thiệu từ Java 9 trở đi với JPMS – Java Platform Module System), "observable module" là một khái niệm dùng để chỉ những module được công khai để các module khác có thể truy cập hoặc phụ thuộc vào chúng.			
				
#✅ Cụ thể, một module "observable" là:

	- Một module đã được đăng ký (resolved) trong thời điểm chạy (runtime) và có thể được quan sát bởi các module khác – tức là:

	- Nó khai báo rõ ràng các module mà nó sử dụng (requires)

	- Nó xuất (exports) các gói mà nó muốn cho module khác dùng


 				
# 📦 Ví dụ đơn giản:

module com.example.util {
    exports com.example.util.tools;
}

	com.example.util là một observable module

	Vì nó exports package com.example.util.tools, các module khác có thể "thấy" và dùng các class trong package này.
			


			
# 💡 So sánh:				
| Loại module           | Có thể bị quan sát không? | Có thể truy cập từ module khác? |
| --------------------- | ------------------------- | ------------------------------- |
| **Observable Module** | ✔ Có                      | ✔ Nếu `exports` được package    |
| **Automatic Module**  | ✔ Có                      | ✔ Tạm thời (dành cho JAR cũ)    |
| **Unnamed Module**    | ❌ Không                   | ❌ Không dùng module-info.java   |
				
				
	
	
# 🧠 Tóm gọn:
	
| Mục                | Nội dung                                                                         |
| ------------------ | -------------------------------------------------------------------------------- |
| 📛 Tên tiếng Nhật  | 観測可能なモジュール                                                                       |
| 🇬🇧 Tên tiếng Anh | Observable Module                                                                |
| 📚 Nghĩa           | Là module mà hệ thống có thể “thấy”, “truy cập” được trong lúc chạy chương trình |
| 🔍 Điều kiện       | Có `exports`, có `requires`, và đã được `resolved` (giải quyết xong phụ thuộc)   |
| 🧱 Dùng trong      | Java Platform Module System (JPMS) – Java 9 trở lên                              |
| 📦 Ví dụ           | `java.base`, `java.sql`, hoặc module do bạn tạo                                  |
