Lesson 187 -- Stream API cơ bản -//

# 1. Stream là gì?
	Stream (ストリーム) trong Java = pipeline xử lý dữ liệu được thiết kế theo functional style.
	
	Thay vì dùng for loop, ta mô tả chuỗi thao tác trên data.

	👉 Senior insight: Stream không phải data structure, nó là view/flow trên data. Dùng để khai báo pipeline, không lưu trữ dữ liệu mới.



# 2. Stream pipeline 
	
	## Gồm 3 phần:
		
		1. Source (ソース) → Collection, Array, I/O, generator.
		2. Intermediate operations (中間操作, chūkan-sōsa) → lazy (chỉ định nghĩa pipeline, chưa chạy).
			map, filter, sorted, distinct, limit…
		3. Terminal operation (終端操作, shūtan-sōsa) → trigger execution.
			forEach, collect, reduce, count, findFirst…


	## Ví dụ pipeline: 

List employees = ...;

List emails = employees.stream()         // source
    .filter(e -> e.getSalary() > 1000)           // intermediate
    .map(Employee::getEmail)                     // intermediate
    .collect(Collectors.toList());               // terminal



# 3. Intermediate vs Terminal – Senior-level

	- Intermediate = “kế hoạch” xử lý, lazy evaluation.
	
	- Terminal = “kích hoạt pipeline”, eager execution.
	
	- Senior cần nhớ: nếu bạn chỉ gọi filter hay map mà không có terminal, không có gì chạy cả.

	employees.stream().filter(e -> e.getSalary() > 1000);
	// Không in gì, vì chưa có terminal



# 4. Enterprise Example
	Case: lấy email nhân viên lương > 1000

List emails = employees.stream()
    .filter(e -> e.getSalary() > 1000)   // lọc theo business rule
    .map(Employee::getEmail)             // transform sang email
    .collect(Collectors.toList());       // gom lại thành List


	## Senior insight: 
	
		- Đây là cách viết “business rule DSL” trong enterprise: code đọc như tiếng Anh, dễ review.
		- filter tương đương Predicate chaining, map tương đương Function chaining.
		- collect đưa kết quả về collection quen thuộc (List/Set/Map).



# 5. Liên hệ thi Silver / Gold

	Silver: thường hỏi output → stream().filter(...).map(...).forEach(...).
	Gold: sẽ xoáy vào:
		Lazy vs eager → tại sao filter không chạy nếu không có terminal.
		Order: map trước filter khác gì filter trước map.
		collect vs reduce.
		Parallel stream (並列ストリーム, heiretsu stream) → multi-thread.



# 6. Senior insight 

	Pipeline trong Stream API = nối Function (map) + Predicate (filter) + Consumer (forEach) → giống như mấy bài trước nhưng chạy trên data set.
	
	Viết code theo kiểu declarative (khai báo muốn làm gì), không imperative (for loop).
	
	Dễ mở rộng business rule: thêm bước mới trong pipeline thay vì viết thêm if-else.
	
	Performance: pipeline chỉ duyệt data một lần (fusion optimization).

# 👉 Tóm lại: Stream API = hiện thực hóa functional pipeline trong Java.

	map = transform (Function).
	
	filter = rule (Predicate).
	
	forEach = side-effect (Consumer).
	
	collect = mang dữ liệu về collection (terminal).
