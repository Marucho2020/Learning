Lesson 181 -- java.util.function -//

# 🔹 1. Function 
	Định nghĩa: nhận một input kiểu T → trả về output kiểu R.
	Method duy nhất: R apply(T t) → đây là SAM interface → có thể dùng lambda.	
	Senior insight:
		Hợp lý với map/transform trong Stream API.
		Có default methods:

			- compose(Function before) → trước khi apply, transform input.
			- andThen(Function after) → chain transform output.


	Ví dụ:	

Function parseLength = s -> s.length();
Function addPrefix = s -> "Len: " + s;

int len = parseLength.apply("Hello"); // 5
String result = parseLength.andThen(Object::toString).andThen(s -> "Len: " + s).apply("Hello"); // "Len: 5"

	Ứng dụng senior:
		- Chaining transformations trong Stream pipelines: map().map().filter().
		- Là foundation cho composable logic trong reactive stream.



# 🔹 2. Consumer

	Định nghĩa: nhận một input → không trả về gì (side-effect).
	Method: void accept(T t)
	Default method: andThen(Consumer after) → chain nhiều action.


Consumer print = System.out::println;
Consumer printUpper = s -> System.out.println(s.toUpperCase());

print.andThen(printUpper).accept("hello");
// Output:
// hello
// HELLO


	## Senior insight: 

	- Dùng để implement event handling, logging, or callbacks trong streams hoặc reactive pipelines.
	- Side-effect aware: cần thận trọng khi dùng trong parallel stream → race condition có thể xảy ra.



# 🔹 3. Supplier
	Định nghĩa: không nhận input → trả về giá trị kiểu T.
	Method: T get()

	Supplier randomSupplier = Math::random;
	System.out.println(randomSupplier.get());


		- Dùng cho lazy evaluation, factory methods, deferred computation.
		- Trong Stream API: Stream.generate(supplier) → tạo infinite stream.
		- 
		- Key pattern trong reactive: cold stream → dữ liệu chỉ được sinh khi có subscription.


# 🔹 4. Predicate

	Định nghĩa: nhận T → trả về boolean.
	
	Method: boolean test(T t)

	Default methods:
		and(Predicate), or(Predicate), negate() → chain logic conditions.

		Predicate nonEmpty = s -> s != null && !s.isEmpty();
		Predicate startsWithA = s -> s.startsWith("A");
		
		boolean result = nonEmpty.and(startsWithA).test("Apple"); // true

	Senior insight:

		- Foundation cho filtering trong Stream API: filter(predicate)
		- Dùng để business rule composition → chain nhiều điều kiện một cách declarative.



# 🔹 5. UnaryOperator và BinaryOperator

	UnaryOperator = Function → nhận T → trả T
	
	BinaryOperator = BiFunction → nhận 2 T → trả T

	## Ví dụ UnaryOperator: 
UnaryOperator square = x -> x * x;
int result = square.apply(5); // 25
		


	## Ví dụ BinaryOperator: 
		
	BinaryOperator sum = Integer::sum;
	int result = sum.apply(3, 7); // 10

	Senior insight:
		Dùng trong reduce, accumulate:
		Stream.of(1,2,3,4).reduce(0, Integer::sum); // BinaryOperator



# 6. Tóm tắt ứng dụng senior

| Interface         | Input | Output  | Use case                                  |
| ----------------- | ----- | ------- | ----------------------------------------- |
| Function\    | T     | R       | Transform data (`map`)                    |
| Consumer       | T     | void    | Side effects, logging, event handlers     |
| Supplier       | -     | T       | Lazy evaluation, factory, Stream.generate |
| Predicate      | T     | boolean | Filtering, rule composition               |
| UnaryOperator  | T     | T       | Map self → reduce transformations         |
| BinaryOperator | (T,T) | T       | Accumulation, reduce, combine             |


# 💡 Senior mindset:

Khi dùng Stream API, hiểu bản chất là pipeline of Functions/Predicates/Consumers.

Biết kết hợp default methods để compose logic thay vì viết nhiều vòng for/if.

Nhận thức side-effect: parallel stream + Consumer → race condition.

Lazy vs eager evaluation: Supplier + Stream.generate.
