# 1. Nhóm 1 – Core tư duy hệ thống (bắt buộc)


	🔹 Control system
	標準化 (standardization)
	一元化 (centralization)
	可視化 (visualization)
	最適化 (optimization)
	自動化 (automation)
	効率化 (efficiency)
	均一化
	👉 Đây là “ngôn ngữ của manager”


	# 🔹 Structure system
	構成 (architecture)
	構造 (structure)
	レイヤー (layer)
	コンポーネント (component)
	モジュール (module)
	依存関係 (dependency)

	👉 Đây là “ngôn ngữ của dev + architect”


	🔹 Flow / Process
	業務フロー (business flow)
	ワークフロー (workflow)
	処理フロー (processing flow)
	承認フロー (approval flow)
	ライフサイクル (lifecycle)

	👉 ServiceNow = sống chết ở đây



# 2. Nhóm 2 – Từ chuyên ServiceNow / ITSM 

	🔹 Core ITSM
		インシデント管理 (incident)
		問題管理 (problem)
		変更管理 (change)
		リクエスト管理 (request)

	👉 4 cái này = xương sống ServiceNow



	🔹 Operation
		SLA（サービスレベル合意）
		エスカレーション
		優先度 (priority)
		影響度 (impact)
		緊急度 (urgency)
	👉 Câu hỏi thi + meeting toàn xoay quanh đây

	🔹 Data / CMDB
		構成管理 (configuration management)
		CMDB
		CI（構成アイテム）
		資産管理 (asset management)
	👉 Đây là phần khó nhất của ServiceNow





# 3. Nhóm 3 – Từ viết tài liệu (cực quan trọng ở Nhật)

	Không biết nhóm này → viết tài liệu như rác.


	🔹 Logic structure
	背景 (background)
	課題 (issue)
	目的 (objective)
	方針 (approach)
	対応 (action)
	効果 (benefit)

	👉 Đây chính là template ngầm của mọi slide


	🔹 Thinking words
	～を実現する
	～を目的とする
	～により
	～に向けて
	～の観点で

	👉 Đây là “grammar của tài liệu IT Nhật”




4. Nhóm 4 – System / Infra (đủ dùng, không cần quá sâu)



	🔹 Basic
		サーバー
		ネットワーク
		DB（データベース）
		API
		クラウド

	🔹 Modern (biết để không bị ngợp)

		マイクロサービス
		API Gateway
		認証 / 認可
		ログ / モニタリング

	👉 Không cần master, nhưng phải hiểu người ta đang nói gì




5. Nhóm 5 – Từ “nguy hiểm” (nghe giống nhau nhưng khác bản chất)

	Đây là chỗ junior chết nhiều nhất.
	
| Từ | Đừng hiểu sai                     |
| -- | --------------------------------- |
| 管理 | không chỉ “manage”, mà là control |
| 運用 | operation (daily running)         |
| 保守 | maintenance                       |
| 監視 | monitoring                        |
| 制御 | control (level cao hơn 管理)        |
