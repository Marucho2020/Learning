Lesson 8 -- ------=ITSM -//

	ITSM = IT Service Management (Quản lý dịch vụ CNTT)

# ① ELI5

	Hiểu thẳng luôn:
		ITSM giống như cách bạn vận hành một quán ăn.

		Khách gọi món → bạn nhận order
		Bếp nấu → xử lý yêu cầu
		Món bị lỗi → phải sửa
		Quán đông → cần quản lý quy trình

	👉 ITSM = cách IT phục vụ người dùng như một “dịch vụ” chứ không phải chỉ code xong là xong



# ② Trong dự án thực tế 

	Trong project ServiceNow ở Nhật, ITSM không phải lý thuyết. Nó là:

	👉 bộ quy trình chuẩn để xử lý mọi thứ liên quan đến IT
	
	Bao gồm các module rất cụ thể:
	
		- Incident（障害対応）
		- Problem（根本原因分析）
		- Change（変更管理）
		- Request（依頼対応）
	Ví dụ thực tế trong dự án:
	
		本システムでは、ITSMプロセスに基づき、
		インシデント管理・変更管理・リクエスト管理の一元化を実現する。

	Trong meeting:
		この対応はIncidentではなくRequestとして扱うべきです。


	User báo:
		“Không login được hệ thống”


	→ Nếu junior:
		Fix luôn


	→ Nếu theo ITSM:
		- Tạo Incident ticket
		- Gán priority
		- Assign team
		- Track SLA
			👉 Không phải sửa nhanh, mà là sửa có quản lý


# ③ Level senior (bản chất)

	Nói thẳng:
		ITSM không phải tool (ServiceNow), mà là operating system của IT organization

	Nó giải quyết vấn đề gì?


		##Chaos (hỗn loạn)
			Ai cũng fix theo cách riêng
			Không trace được lịch sử
			
		##Không scale được
			10 user ok, 10,000 user → chết
		
		##Không đo được
			Bao lâu fix xong?
			Team nào bottleneck?

		👉 ITSM biến IT từ:
			“thợ sửa máy”
			→ thành
			“dịch vụ có SLA, KPI, process”
	


	## Nếu không có ITSM? 

		- Ticket bị bỏ quên
		- Bug fix trùng lặp
		- Không biết ai chịu trách nhiệm
		- Release gây lỗi production liên tục

	👉 Nói ngắn gọn:
		Không có ITSM = hệ thống sẽ chết vì chính team vận hành nó




# Liên kết với các concept khác:

		標準化（Standardization）
			→ Chuẩn hóa cách xử lý Incident
			
		一元化（Centralization）
			→ Gom tất cả ticket vào 1 hệ thống (ServiceNow)
			
		可視化（Visualization）
			→ Dashboard SLA, trạng thái ticket
			
		自動化（Automation）
			→ Auto assign, auto escalate

		👉 ITSM là nền, mấy cái trên là hệ quả




# Ví dụ tài liệu

	本プロジェクトでは、ITSMフレームワークに基づき、
	インシデント管理、問題管理、変更管理の標準化および一元化を実施する。

	また、SLA遵守率向上のため、
	ワークフローの自動化および可視化を行う。


# Slide


	ITSM導入目的
		- 運用プロセスの標準化
		- 対応状況の可視化
		- SLA遵守率の向上
	
	対象プロセス
		- Incident管理
		- Problem管理
		- Change管理
		- Request管理
	
	効果
		- 対応品質の均一化
		- 属人化の排除
		- 運用効率の向上



# Meeting phrase

	ITSMプロセスに乗せて対応しないと、後でトラッキングできなくなります。


# So sánh & bẫy 

	## 1. ITSM vs 運用管理 
	
|          | ITSM             | 運用管理      |
| -------- | ---------------- | --------- |
| Bản chất | Framework        | Hoạt động |
| Mức độ   | Strategic        | Tactical  |
| Ví dụ    | Incident process | Fix bug   |
	
	
	
	👉 Junior hay nhầm:	
		“Làm vận hành” = ITSM
		→ Sai. ITSM là cách vận hành, không phải hành động.	
			
			
	
	## 2. ITSM vs ServiceNow
	
		👉 Sai lầm cực phổ biến:

			❌NG: “Chúng ta dùng ServiceNow nên đã có ITSM”
			✅OK: “ServiceNow chỉ là tool để implement ITSM”

		👉 Tool ≠ Process	
			
	
	
	## 3. Incident vs Request
	
		Junior hay làm:
			User hỏi reset password → tạo Incident ❌

		Correct:
			Reset password → Request ✅
			System down → Incident ✅	
		
	
	
	
# Mindset senior


	## 1. Không nghĩ “fix”, mà nghĩ “flow”

	Senior sẽ nghĩ:
		Ticket này thuộc process nào?
		Có SLA không?
		Có cần escalation không?



	##2. Mọi thứ phải trace được
		Nếu không track được → coi như chưa làm




	## 3. Không optimize local, mà optimize system

		Junior:
			Fix nhanh cho xong

		Senior:
			Fix theo process để cả system vận hành ổn định




# 4. Khi nào dùng từ ITSM?

		Xuất hiện nhiều trong:

			案件概要（giới thiệu hệ thống）
			方針（policy / direction）
			運用設計（operation design）

		👉 Không dùng trong code-level discussion

		“ITSM対応してください”
		“ITSMプロセスに従う”

	👉 Hiểu ngay:
