Lesson 17 -- Wildcard * -//

	Wildcard = ký tự đại diện *

	👉 Nghĩa là:
		“áp dụng rule này cho TẤT CẢ field (hoặc nhiều object cùng lúc)”


		
		
		
# 	Trong ACL nó xuất hiện ở đâu?
		
	Trong field của ACL:
		Table: incident
		Field: * ← wildcard

	👉 nghĩa là:
		incident.*.read
		→ đọc toàn bộ field của bảng incident		
				
			
		
#	🔥 3. So sánh cho rõ 
		
	##Không wildcard:	
		incident.short_description.read
		→ chỉ apply cho field short_description		
		
		
		
	## Có wildcard: 		
		incident.*.read
		→ apply cho mọi field trong incident		
			
		
		
💣 4. Vấn đề nguy hiểm (đây mới là thứ quan trọng)		
		
		
	Wildcard = scope cực rộng

	👉 nó có thể:
		override các rule nhỏ hơn
		gây conflict khó debug		
			
		
		
		
# ⚔️ 5. Case thực tế 		
		
		
	Bạn tạo:
		Rule 1:
			incident.*.read → allow cho role IT
		
		
		Rule 2:
			incident.salary.read → deny
		
		
		
		Kết quả:

		👉 user IT:
			đọc được mọi thứ ✅
			nhưng KHÔNG đọc được salary ❌

			→ vì deny field cụ thể thắng		
		
		
# 🧨 6. Sai lầm phổ biến		
		
		
	❌ Junior nghĩ:
		wildcard = tiện → dùng cho nhanh
		→ Sai.		
			
		
		
	❌ Hậu quả:
		rule chồng chéo
		không biết rule nào đang apply
		debug = ác mộng	
		
		
		
# 🧠 7. Bản chất hệ thống 		
		
	Wildcard không phải là “syntax tiện lợi”
	👉 nó là:
	“mass rule injection vào toàn bộ field”		
			
		
		
		
# 🔬 8. Cách engine hiểu wildcard		
		
		
	incident.*.read	
		
		
	→ internally =	
		
		incident.field1.read
		incident.field2.read
		incident.field3.read
		...		
				
	👉 tức là nó expand logic ra toàn bộ field	
		
		
# ⚠️ 9. Interaction với thứ khác 


	Wildcard sẽ interact với:
		deny rule
		field-specific rule
		parent table ACL
		👉 tạo ra behavior rất khó đoán nếu không hiểu order



# 🧠 10. Tư duy senior 


	Đừng hỏi:
		“Wildcard là gì?”

	Hỏi:
		“Wildcard này đang ảnh hưởng tới bao nhiêu field và rule nào nó override?”



		
# 🔥 11. Khi nào nên dùng wildcard 		
		
		
	✔ OK khi:
		define default rule cho table
		setup baseline security	
				
		
		
#🚫 12. Khi KHÔNG nên dùng	
		
		❌ Khi:

		có nhiều field sensitive
		system đã có nhiều ACL
		team đông người (dễ conflict)		
				
		
		
# 💥 13. Cách nhớ chuẩn 		
		
	* = toàn bộ
	dùng sai = mở toang cửa hoặc khóa nhầm tất cả		
		
		
		
# 🚨 14. Kết luận thẳng 		
		
	Wildcard giúp nhanh → nhưng mất kiểm soát
	Là nguyên nhân chính gây bug ACL
	Không hiểu rõ → đừng dùng
