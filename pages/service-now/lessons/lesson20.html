# CSDM chia tầng kiểu này:


Business Layer
   ↓
Business Service (VD: Internet Banking)

Application Layer
   ↓
Application Service (VD: Payment API)

Technical Layer
   ↓
Application / DB / Server

Infrastructure Layer
   ↓
VM / Network / Storage





# 🧩 Breakdown thực tế:
	1. Business Service
		thứ business hiểu
		VD: "Mobile Banking"
		
	2. Application Service
		logic backend
		VD: "Auth Service", "Payment Service"
	
	3. Application / DB
		microservice, database
	
	4. Infrastructure
	
		EC2 / VM / Kubernetes node
		
		
	🔥 Relationship chuẩn:
	
		Business Service
		   └── depends on → Application Service
				 └── runs on → Application
					   └── hosted on → Server

	👉 Đây là xương sống CMDB
		Sai 1 link → impact analysis sai toàn bộ





# CMDB (Configuration Management Database)
	→ DB quản lý toàn bộ tài sản IT + quan hệ


# CI (Configuration Item) 


	bất kỳ thành phần nào trong hệ thống:
		server
		app
		API
		network


# Relationship
	→ quan hệ giữa CI 



# Dependency
	→ sự phụ thuộc (A chết → B chết) 



# Upstream / Downstream 

	upstream = phía trước (nguồn)
	downstream = phía sau (phụ thuộc)


# MID Server 
	→ agent trung gian để ServiceNow scan network nội bộ



# Discovery

	→ auto scan infra 


# Service Mapping

	→ map service theo business flow



# CSDM

	→ chuẩn thiết kế CMDB của ServiceNow 


# Reconciliation Rules (Quy tắc hòa giải)

	→ rule quyết định:
	data nào là “đúng” khi có nhiều nguồn 


# Audit Template
	→ rule kiểm tra data CMDB có hợp lệ không

		Họ build CMDB như Excel → thất bại

			nghĩ CMDB như graph + system model
