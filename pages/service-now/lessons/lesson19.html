Lesson 19 -- Import Data -//

#  1. Bức tranh tổng thể (nếu hiểu sai chỗ này → hỏng hết) 

External Data (CSV / Excel / API)
        ↓
Import Set Table (staging)
        ↓
Transform Map (logic)
        ↓
Target Table (production data)




# 2. Sai lầm phổ biến (junior hay mắc) 

Import thẳng vào target table → sai
Không dùng coalesce → duplicate nổ tung
Mapping thủ công không kiểm soát → data inconsistency
Không validate data → garbage in → garbage out






# 3. Import Set Table (staging layer) 

	👉 Bản chất:

		Là bảng tạm (temporary table)
		Không dùng cho business logic
		Chỉ để chứa raw data

	👉 Tư duy senior:

		Xem nó như “data lake mini”
		Không trust dữ liệu ở đây
		Không join business logic vào đây



# 4. Transform Map = nơi bạn thực sự code logic 


	👉 Đây mới là “trái tim” của bài toán.

	Transform map làm 3 việc:
		Mapping field (simple)
		Data transformation (format, normalize)
		Decision logic (insert/update/ignore)


# 5.Coalesce = thứ quyết định hệ thống sống hay chết 

	👉 Hiểu cực kỳ rõ:

		Import 1 → 100 records
		Import lại → thêm 100 records nữa (duplicate)


	👉 Có coalesce:
		Import 1 → 100 records
		Import lại → update 100 records


| Junior                  | Senior                   |
| ----------------------- | ------------------------ |
| Coalesce là option      | Coalesce là bắt buộc     |
| Import là copy data     | Import là sync data      |
| Không nghĩ về re-import | Thiết kế cho idempotency |


# 6. Các loại Coalesce (hiểu sâu) 


	1. Single-field

	Ví dụ:
		email
		serial_number

	👉 dùng khi field đó unique thật


	2. Multiple-field
		first_name + last_name + dob
		👉 dùng khi không có unique key




	3. Conditional (script)
			
		if (source.status == "active") {
			answer = true;
		} else {
			answer = false;
		}	
			
	
		👉 dùng khi:

		data không sạch
		rule phức tạp
		cần business logic	
			
	
# 	7. Mapping
	
	
	❌ Không phải:

		copy field A → field B
			✅ Thực tế:
			normalize data
			convert format
			enrich data  (làm giàu dữ liệu - thêm thông tin vào cái ban đầu cho nó đầy đủ hơn )
			validate
	
	
	Ví dụ thực tế:
		phone: "090-123-4567"
			→ remove "-"
			→ chuẩn hóa format
		
	
	
# 8. Transform Map nâng cao 	(senior mindset)



	Bạn nên làm thêm:
		onBefore script → validate
		onAfter script → trigger logic
		transform script → xử lý từng field


# 9. Data Quality (điểm mà người mới xem nhẹ)


	👉 Nếu dữ liệu đầu vào bẩn:
		duplicate
		missing field
		sai format

	👉 thì:
		system bạn sẽ chết dần theo thời gian



	## 🧠 Senior sẽ làm gì? 

		validate trước import
		clean data trong transform
		reject bad data


# 10. Re-import strategy (rất quan trọng)


	Bạn phải assume(cho rằng):

		dữ liệu sẽ được import lại nhiều lần

	👉 Vì vậy:
		coalesce là bắt buộc
		logic phải idempotent (= chạy 1 lần hay 100 lần → kết quả vẫn như nhau, không bị “phình” thêm tác dụng phụ.)



# 11. Cleanup


	Sau khi transform:
		xóa import set cũ (nếu cần)
		audit log
		kiểm tra record lỗi
