Lesson 18 -- Scope -//

# Nguyên tắc Best practice thực chiến
1. Logic thuộc app nào thì để trong scope app đó.
2. App khác muốn dùng thì expose qua Script Include/API rõ ràng.
3. Hạn chế cho app khác ghi trực tiếp table.
4. Đừng viết business logic mới trong Global nếu không bắt buộc.
5. Cross-scope privilege phải tối thiểu, không mở bừa.
6. Table quan trọng nên dùng Caller Restriction.
7. Đừng sửa app OOTB tùy tiện; ưu tiên extension/configuration.



# Cách debug khi bị lỗi scope 


1. Xác định code đang chạy trong scope nào.
2. Xác định resource thuộc scope nào.
3. Kiểm tra resource có Accessible from = All application scopes không.
4. Kiểm tra Application Cross-Scope Access.
5. Xem record trong sys_scope_privilege.
6. Nếu gọi Script Include, kiểm tra class name có prefix đúng không.
7. Nếu là table, kiểm tra table Application Access.



# Tóm lại 
Scope = biên giới của app trong ServiceNow.
Global = khu chung, mạnh nhưng nguy hiểm.
Scoped App = app riêng, có namespace riêng.
Cross-scope = app này gọi app khác.
sys_scope_privilege = bảng ghi quyền cross-scope.
Accessible from = resource có cho app khác thấy không.
Caller Access = kiểm soát/track/siết lời gọi từ app khác.


Scope không phải phiền phức.
Scope là hàng rào chống app này phá app kia.
